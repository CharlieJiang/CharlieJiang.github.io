Java内部类

发表于 2020-11-26 分类于 Java ， Java基础 阅读次数：

什么是内部类

在一个类的内部定义的类叫做内部类。

内部类种类

  内部类可分为静态内部类和非静态内部类。非静态内部类又可以分为成员内部类、局部内部类、匿名内部类。

  1. 静态内部类

    使用static修饰的内部类叫做静态内部类。

    • 静态内部类的创建无需依赖于外部类 
      public class AClass {

    static class StaticInnerClass{ //静态内部类
        public void print(){
            System.out.println("StaticInnerClass");
        }
    }

    public static void main(String[] args) {
        StaticInnerClass sic = new StaticInnerClass(); //直接创建静态内部类实例
        sic.print();
    }

}
    • 静态内部类无法使用外部类的非static属性和方法 
      public class AClass {

    private String name;
    public static int age;

    private void printName() {
        System.out.println(name);
    }

    public static void printAge() {
        System.out.println(age);
    }

    static class StaticInnerClass{ //静态内部类
        public void print(){
            name = "cocoas"; //编译错误
            printName(); //编译错误
            age = 29; //调用外部类静态属性
            printAge(); //调用外部类静态方法
        }
    }
}

  2. 成员内部类

    成员内部类是普通的内部类，其属于外部类的成员，可以无限制访问外部类的全部成员属性和成员方法，即使是私有的。

  成员内部类示例： 

public class AClass {

    private String name;
    public static int age;

    private void printName(){
        System.out.println(name);
    }

    public static void printAge(){
        System.out.println(age);
    }

    class FieldInnerClass { //成员内部类
        public void print(){
            name = "cocoas"; //调用外部类私有属性
            printName(); //调用外部类私有方法
            age = 29; //调用外部类静态属性
            printAge(); //调用外部类静态方法
        }
    }

    public static void main(String[] args) {
        AClass ac = new AClass();
        FieldInnerClass fic = ac.new FieldInnerClass();
        fic.print();
    }
}

  注意事项：

  • 成员内部类中不能存在任何的static方法和变量,即成员内部类中不能定义自己的静态域。 
    class FieldInnerClass { //成员内部类
    private static int year; //编译错误
    public static void printYear(){} //编译错误
}
  • 成员内部类依附于外部类，只有先创建外部类对象才能创建成员内部类。
    public static void main(String[] args) {
    AClass ac = new AClass(); //先创建外部类
    FieldInnerClass fic = ac.new FieldInnerClass(); //使用外部类对象创建成员内部类
    fic.print();
}
  1. 局部内部类

    局部内部类是指定义在方法或代码块内的类。
    局部内部类并不常用。

  局部内部类与成员内部类的区别在于，局部内部类的访问只限于方法或代码块内。

  • 方法中的局部内部类： 
    public PartInnerClass partInnerClass; //编辑错误
private void printName() {
    System.out.println(name);
    class PartInnerClass{ //方法中的局部内部类
        public void print(){
            System.out.println("PartInnerClass:" + name);
        }
    }
    PartInnerClass pic = new PartInnerClass(); //创建局部内部类
    pic.print(); //使用局部内部类
}
  • 代码块中的局部内部类： 
    public PartInnerClass2 partInnerClass2; //编辑错误
{
    class PartInnerClass2{ //代码块中的局部内部类
        public void print(){
            System.out.println("PartInnerClass2:" + name);
        }
    }
    PartInnerClass2 pic = new PartInnerClass2(); //创建代码块局部内部类
    pic.print(); //使用代码块局部内部类
}

  1. 匿名内部类

    匿名内部类是指没有名字的内部类。
    匿名内部类之所以没有名字，是因为其是唯一一种没有构造器的类。匿名内部类在编译的时候由系统自动起名为Outer$1.class

    • 匿名内部类一般用于接口回调，如事件响应
      public interface ClickListener{
    public void onClick();
}

public void click(int position){
    new ClickListener(){ //匿名内部类
        @Override
        public void onClick() {
            System.out.println("click position " + position);
        }
    };
}

内部类的特性

  • 内部类可以拥有多个实例，每个实例都有自己的状态信息，且与其他外部对象的信息相互独立。
  • 在外部类中，可以让多个内部类以不同的方式实现相同接口或继承同一个类。
  • 内部类对象创建的时间点并不依赖于外部类对象的创建。
  • 内部类提供了更好的封装，除了其外部类外，其他类都无法访问。
  • 内部类能够实现类的多继承。每个内部类都能够独立地继承一个其他类，一个类中若存在多个内部类，则可以实现类的多继承。

内部类与外部类的关联

  1. 创建非静态内部类对象需要借助于外部类，静态内部类则不需要 

    public class AClass {

 class FieldInnerClass { //成员内部类
 }

 public static void main(String[] args) {
     AClass ac = new AClass();
     FieldInnerClass fic = ac.new FieldInnerClass();
     fic.print();
 }
}

  2. 内部类一般会持有外部类的引用，所以内部类可以调用外部类的属性和方法，即使是私有的。

  3. 内部类只是个编译时的概念，一旦编译成功，内部类与外部类就是完全不同的两个类（两个类的关系还在）。
