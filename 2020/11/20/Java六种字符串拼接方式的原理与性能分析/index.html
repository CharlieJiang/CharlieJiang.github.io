Java六种字符串拼接方式的实现原理与性能分析

发表于 2020-11-20 更新于 2020-11-26 分类于 Java 阅读次数：

Java 常见字符串拼接方式

Java常见的字符串拼接方式有如下六种：

  • 通过“+”运算符拼接
  • 通过StringBuffer拼接
  • 通过StringBuilder拼接
  • 通过String.concat()方法拼接
  • 通过org.apache.commons.lang.StringUtils.join()方法拼接
  • 通过String.join()方法拼接 （Java 1.8新增）

各字符串拼接方式的原理及性能分析

下面分别分析这六种拼接方式的原理及性能

1. “+”运算符拼接

  “+”拼接方式背后是通过StringBuilder实现的，每次拼接都会重新创建新的字符串对象。具体原理请查看
  Java字符串”+”方式拼接的实现原理

2. StringBuffer拼接

  StringBuffer通过append()方法实现字符串拼接

  StringBuffer.append() 源码如下：

public synchronized StringBuffer append(String str) {
    toStringCache = null;
    super.append(str);
    return this;
}

  其中super.appen()源码如下：

public AbstractStringBuilder append(String str) {
    if (str == null)
        return appendNull();
    int len = str.length();
    ensureCapacityInternal(count + len);
    str.getChars(0, len, value, count);
    count += len;
    return this;
}

  可以看到：StringBuffer的每次拼接都是在其内部char[]数组的基础上进行扩容，以存放待拼接的字符串。

  只有当我们主动调用StringBuffer.toString()方法时，StringBuffer才会将拼接的字符串以创建新字符串对象的方式返回。

  StringBuffer.toString()的源码如下：

public synchronized String toString() {
    if (toStringCache == null) {
        toStringCache = Arrays.copyOfRange(value, 0, count);
    }
    return new String(toStringCache, true);
}

3. StringBuilder拼接

  StringBuilder是JDK1.5之后才加入的，其字符串拼接的方式与StringBuffer类似，只是相关的拼接方法append()、toString()等都缺少了synchronized关键字，即StringBuilder是非线程安全的。在单线程的情况下，两者相差不大，StringBuilder略优于StringBuffer。

  StringBuilder.append()源码如下：

public StringBuilder append(String str) {
    super.append(str);
    return this;
}

性能比较：StringBuilder>StringBuffer

4. String.concat()方法拼接

  concat()方法用于将参数字符串连接到原字符串末尾。方法源码如下：

public String concat(String str) {
    int otherLen = str.length();
    if (otherLen == 0) {
        return this;
    }
    int len = value.length;
    char buf[] = Arrays.copyOf(value, len + otherLen);
    str.getChars(buf, len);
    return new String(buf, true);
}

  从源码可以看到，concat()的实现类似于StringBuffer.appen()方法，都是通过扩容原char[]数组的方式存放新的字符串，区别是concat()方法每次都会生成新的字符串对象，对性能影响较大。concat()又与“+”拼接方式类似，只是不会频繁创建StringBuilder对象。

  所以concat()方法的性能应该与“+”方式类似，应该是相同数量级的，concat()性能稍强于“+”。由于频繁创建String，concat()性能是远弱于StringBuffer的。

性能比较：StringBuffer>>concat()>”+”

5. StringUtils.join()方法拼接

  StringUtils.join()一般用于将数组或集合与指定分隔符拼接到一起，形成新的字符串。

  StringUtils.join()源码如下：

public static String join(final Object[] array, final String separator) {
    if (array == null) {
        return null;
    }
    return join(array, separator, 0, array.length);
}
public static String join(final Object[] array, String separator, final int startIndex, final int endIndex) {
    if (array == null) {
        return null;
    }
    if (separator == null) {
        separator = EMPTY;
    }

    // endIndex - startIndex > 0:   Len = NofStrings *(len(firstString) + len(separator))
    //           (Assuming that all Strings are roughly equally long)
    final int noOfItems = endIndex - startIndex;
    if (noOfItems <= 0) {
        return EMPTY;
    }

    final StringBuilder buf = newStringBuilder(noOfItems);

    if (array[startIndex] != null) {
        buf.append(array[startIndex]);
    }

    for (int i = startIndex + 1; i < endIndex; i++) {
        buf.append(separator);

        if (array[i] != null) {
            buf.append(array[i]);
        }
    }
    return buf.toString();
}

  从源码可以看到，StringUtils.join()是通过newStringBuilder()创建StringBuilder对象，循环追加集合中字符串和分隔符完成拼接，最后调用StringBuilder.toString()返回结果。

  再看下newStringBuilder()方法：

private static StringBuilder newStringBuilder(final int noOfItems) {
    return new StringBuilder(noOfItems * 16);
}

  newStringBuilder()方法传入的是待拼接字符串集合的size，其内部创建了一个初始化长度为 size * 16 的StringBuilder()对象，使得拼接字符串时能极大的减少StringBuilder内部char[]数组扩容的次数，从而提供性能。

  StringUtils.join()相较于StringBuilder只是每次多追加了一个分隔符，所以性能应该是在同一个数量级。

性能比较：StringUtils.join()<StringBuilder

6. String.join()方法拼接

  String.join()是JDK1.8新加入的静态方法，其功能与StringUtils.join()相同，都是将集合中元素与指定分隔符拼接成新的字符串。两者的实现方式也相似。

  String.join()的源码如下：

public static String join(CharSequence delimiter,
        Iterable<? extends CharSequence> elements) {
    Objects.requireNonNull(delimiter);
    Objects.requireNonNull(elements);
    StringJoiner joiner = new StringJoiner(delimiter);
    for (CharSequence cs: elements) {
        joiner.add(cs);
    }
    return joiner.toString();
}

  String.join()的源码中使用了StringJoiner，通过循环调用其add()方法来完成字符串的拼接。StringJoiner也是JDK1.8新加入的类，它是专门用来拼接带分隔符的字符串的。

  StringJoiner.add()方法源码如下：

public StringJoiner add(CharSequence newElement) {
    prepareBuilder().append(newElement);
    return this;
}

  add()方法中通过prepareBuilde()方法创建StringBuilder对象并调用StringBuilder.append()方法拼接字符串，再看下prepareBuilde()方法：

private StringBuilder prepareBuilder() {
    if (value != null) {
        value.append(delimiter);
    } else {
        value = new StringBuilder().append(prefix);
    }
    return value;
}

  此时就一目了然了：prepareBuilder()方法主要就是向外提供StringBuilder对象，并且提前在StringBuilder中追加了一个字符串的分隔符，这样在调用add()方法时就无需再传入delimiter参数了。
  通过比较StringUtils.join()和String.join()可以发现，两者的实现方式基本一致，都是通过StringBuilder+循环迭代实现对集合元素的拼接。区别就在于前者是自己调用StringBuilder，而后者是通过StringJoiner调用StringBuilder。所以，两者的性能是基本相同的。不过StringUtils.join()在创建StringBuilder对象时预设了初始容量为集合size * 16，由于StringBuilder的默认容量是16，这样做显然会减少StringBuilder扩容的次数，从而一定程度上提高了性能。

  StringBuilder的构造函数：

public StringBuilder() {
    super(16);//StringBuilder的默认容量
}

性能比较：String.join()<StringUtils.join()

总结

  综上所述，六种字符串拼接方式的性能高低如下：

StringBuilder>StringBuffer>StringUtils.join()>String.join()>>String.concat()>”+”

  性能测试代码

package com.cocoas.pojo;

import org.apache.commons.lang3.StringUtils;

import java.util.ArrayList;
import java.util.List;

public class MainClass {

    public static void main(String[] args) {
        MainClass mc = new MainClass();
        List<String> list = mc.createStringList(10000);
        mc.printTime("+",mc.plusAppendStringTime(list));
        mc.printTime("StringBuffer",mc.stringBufferAppendStringTime(list));
        mc.printTime("StringBuilder",mc.stringBuilderAppendStringTime(list));
        mc.printTime("String.concat",mc.stringConcatAppendStringTime(list));
        mc.printTime("StringUtils.join",mc.stringUtilsJoinAppendStringTime(list));
        mc.printTime("String.join",mc.stringJoinAppendStringTime(list));
    }

    /**
     * 创建测试字符串列表：供拼接
     * @param count 需要创建的字符串数量
     * @return
     */
    public List<String> createStringList(int count){
        List<String> list = new ArrayList<>();
        while(count > 0){
            list.add("cocoas_" + count--);
        }
        return list;
    }

    /**
     * "+"方式拼接字符串的时间计算
     * @param strings
     * @return
     */
    public long plusAppendStringTime(List<String> strings){
        long startTime = System.currentTimeMillis();
        String result = "";
        for(String str : strings){
            result += str;
        }
        long endTime = System.currentTimeMillis();
        return endTime - startTime;
    }

    /**
     * StringBuffer方式拼接字符串的时间计算
     * @param strings
     * @return
     */
    public long stringBufferAppendStringTime(List<String> strings){
        long startTime = System.currentTimeMillis();
        StringBuffer sb = new StringBuffer();
        for(String str : strings){
            sb.append(str);
        }
        sb.toString();
        long endTime = System.currentTimeMillis();
        return endTime - startTime;
    }

    /**
     * StringBuilder方式拼接字符串的时间计算
     * @param strings
     * @return
     */
    public long stringBuilderAppendStringTime(List<String> strings){
        long startTime = System.currentTimeMillis();
        StringBuilder sb = new StringBuilder();
        for(String str : strings){
            sb.append(str);
        }
        sb.toString();
        long endTime = System.currentTimeMillis();
        return endTime - startTime;
    }

    /**
     * String.concat方式拼接字符串的时间计算
     * @param strings
     * @return
     */
    public long stringConcatAppendStringTime(List<String> strings){
        long startTime = System.currentTimeMillis();
        String result = "";
        for(String str : strings){
            result = result.concat(str);
        }
        long endTime = System.currentTimeMillis();
        return endTime - startTime;
    }

    /**
     * org.apache.commons.lang.StringUtils.join()方式拼接字符串的时间计算
     * @param strings
     * @return
     */
    public long stringUtilsJoinAppendStringTime(List<String> strings){
        long startTime = System.currentTimeMillis();
        String result = "";
        result = StringUtils.join(strings);
        long endTime = System.currentTimeMillis();
        return endTime - startTime;
    }

    /**
     * String.join方式拼接字符串的时间计算
     * @param strings
     * @return
     */
    public long stringJoinAppendStringTime(List<String> strings){
        long startTime = System.currentTimeMillis();
        String result = "";
        result = String.join("",strings);
        long endTime = System.currentTimeMillis();
        return endTime - startTime;
    }

    /**
     * 打印方法耗时
     * @param functionName  方法名
     * @param timeInMillis  时间：毫秒
     */
    public void printTime(String functionName,long timeInMillis){
        System.out.println(functionName + ":" + timeInMillis + "ms");
    }

}
