Java六种字符串拼接方式的实现原理与性能分析

发表于 2020-11-20 更新于 2020-11-26 分类于 Java 阅读次数：

Java 常见字符串拼接方式

Java常见的字符串拼接方式有如下六种：

  • 通过“+”运算符拼接
  • 通过StringBuffer拼接
  • 通过StringBuilder拼接
  • 通过String.concat()方法拼接
  • 通过org.apache.commons.lang.StringUtils.join()方法拼接
  • 通过String.join()方法拼接 （Java 1.8新增）

4. String.concat()方法拼接

  concat()方法用于将参数字符串连接到原字符串末尾。方法源码如下：

  从源码可以看到，concat()的实现类似于StringBuffer.appen()方法，都是通过扩容原char[]数组的方式存放新的字符串，区别是concat()方法每次都会生成新的字符串对象，对性能影响较大。concat()又与“+”拼接方式类似，只是不会频繁创建StringBuilder对象。

  所以concat()方法的性能应该与“+”方式类似，应该是相同数量级的，concat()性能稍强于“+”。由于频繁创建String，concat()性能是远弱于StringBuffer的。

性能比较：StringBuffer>>concat()>”+”

  再看下newStringBuilder()方法：

  newStringBuilder()方法传入的是待拼接字符串集合的size，其内部创建了一个初始化长度为 size * 16 的StringBuilder()对象，使得拼接字符串时能极大的减少StringBuilder内部char[]数组扩容的次数，从而提供性能。

  StringUtils.join()相较于StringBuilder只是每次多追加了一个分隔符，所以性能应该是在同一个数量级。

性能比较：StringUtils.join()<StringBuilder

总结

  综上所述，六种字符串拼接方式的性能高低如下：

StringBuilder>StringBuffer>StringUtils.join()>String.join()>>String.concat()>”+”
