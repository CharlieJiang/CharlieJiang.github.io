Java泛型

发表于 2020-11-28 分类于 Java ， Java基础 阅读次数：

引入时间

JDK 1.5

什么是泛型

泛型就是以类型参数的方式限定数据的类型。

泛型中使用<类型>来声明数据的类型，例如 ArrayList<String>表明集合元素是字符串

为什么使用泛型

  在泛型出现之前，通用程序的设计一般是通过继承来实现的。
  例如ArrayList就是通过Object[]数组实现的，因Object是所有类的基类，所以ArrayList就可以存放众多数据类型的数据。但这种方式存在的问题是，我们无法阻止不同类型的数据被插入到相同的ArrayList中，而且每次从ArrayList中提取数据都必须进行强制类型转换。这就很容易发生类型转换异常，更麻烦的是，这种异常并不是发生在编译阶段，而是在运行阶段。
  我们所期望的是，程序的异常或错误，应该在编译阶段就暴露出来。

泛型的优点

  • 使代码具有更好的可读性。我们看一眼就知道集合存储的元素类型。
  • 使程序具有更好的安全性。使用泛型时，编译器会自动帮我们检查插入集合数据的正确性，避免向集合中插入错误的数据类型。

泛型语法

  1. 泛型类

    语法：在类名后添加类型参数的声明，类型参数声明部分由尖括号<>以及尖括号中指定的类型组成，尖括号中若存在多个类型，以逗号分隔。

    class GenericClass<T1,T2,...,Tn>{ }

  2. 泛型接口

    语法：与泛型类相似

    interface GenericInterface<T1,T2,...,Tn>{
    T1 method1();
    T2 method2();
}

      实现泛型接口的两种方式：

    • 实现泛型接口的类明确声明泛型类型 
      public class GenericInterfaceImpl implements GenericInterface<Integer,String> { //声明泛型类型为Integer和String
    @Override
    public Integer method1() {
        return Integer.valueOf(123);
    }

    @Override
    public String method2() {
        return "abc";
    }

    public static void main(String[] args) {
        GenericInterfaceImpl gii = new GenericInterfaceImpl();
        System.out.println(gii.method1() + ":" + gii.method2());
    }
    
}
    • 实现泛型接口的类不明确声明泛型类型 
      class GenericInterfaceImpl<T1,T2> implements GenericInterface<T1,T2>{ //未明确声明泛型类型
    @Override
    public T1 method1() {
        return null;
    }

    @Override
    public T2 method2() {
        return null;
    }
}

  3. 泛型方法

    语法：与泛型类和泛型接口不同，泛型方法中类型参数的声明放在方法的返回值类型之前。

泛型方法中，类型参数可以用来声明方法返回值类型，并且能作为泛型方法实际类型参数的占位符。 

修饰符 参数类型 返回值类型 方法名（形参）{方法体}

public <T> T genericMethod(T obj) {
    return obj;
}

  省略类型参数

  • 在泛型类或泛型接口中，定义的非静态泛型方法可以省略类型参数。
    public interface GenericInterface<T> { //泛型接口

    public T getValue(); //非静态泛型方法：可以省略参数类型<T>

    public static <T> T getStaticValue(T t){    //静态泛型方法：无法省略参数类型<T>
        return t;
    }

}

public class GenericClass<T> {  //泛型类

    public T method(T obj){ //非静态泛型方法：可以省略参数类型<T>
        return obj;
    }

    public static <T> T staticMethod(T obj){    //静态泛型方法：无法省略参数类型<T>
        return obj;
    }

}
  • 不过，在一般类或接口中，定义的泛型方法则无法省略参数类型。
    public class GenericMethodClass {   //一般类

    public <T> T genericMethod(T obj) { //非静态泛型方法：无法省略参数类型<T>
        return obj;
    }

    public static <T> void printClassType(T obj){   //静态泛型方法：无法省略参数类型<T>
        System.out.println(obj.getClass().toString());
    }
}
      类型参数推断（Type Argument Reference）
      调用泛型方法时，一般不需要指明参数类型，因为编译器会为我们找出具体的类型。这种行为称为类型参数推断。
      类型参数推断示例：
    public class GenericMethodClass {  

    public static <T> void printClassType(T obj){ 
        System.out.println(obj.getClass().toString());
    }

    public static void main(String[] args) {
        printClassType("cocoas");
        printClassType(123);
    }

}

//输出结果：
class java.lang.String  //自动推断为String类型
class java.lang.Integer //自动推断为Integer类型

泛型原理：类型擦除

Java中引入泛型是为了在编译时提供更严格的类型检查，并支持泛型编程。
不同于C++的模板机制，Java泛型是使用类型擦除实现的。在使用泛型时，任何具体的类型信息都被擦除了。
伪泛型：Java中的泛型基本上都是在编译器这个层次来实现的，在编译期间，所有的泛型信息都会被擦除，编译后的泛型都会变成原始类型。
原始类型：原始类型就是泛型类型在擦除了泛型信息后，在字节码中真正的类型。原始类型的名字就是删除类型参数后的泛型类型的类名。如List的原始类型就是List。

  类型擦除过程：

  • 替换泛型的类型参数
    把泛型中所有的类型参数替换为Object，如果指定了类型边界，则使用类型边界替换。
    因此，生成的字节码中紧紧包含普通的类、接口和方法。
  • 擦除泛型的类型声明
    去除尖括号<>这部分声明内容。
    如 T method()擦除后为Object method()、List擦除后为List。
    如有必要，擦除后会插入类型转换，以保证类型安全。
  • 生成桥接方法以保留扩展泛型类型中的多态性
    类型擦除确保不会为参数化类型创建新类。因此，泛型不会产生运行时开销。

  比如定义一个整型列表List<Integer>和一个字符串型列表List<String>，在编译时，两者都会被擦除成List类型。所以会有List<Integer>通过getClass()方法返回的类型信息与List<String>的相同。

ArrayList<String> arrayString=new ArrayList<>();
ArrayList<Integer> arrayInteger=new ArrayList<>();
System.out.println(arrayString.getClass()==arrayInteger.getClass());

//输出结果为：
true

类型边界

类型边界可以对泛型的类型参数设置限制条件。

  声明语法： 

<类型参数名称 extends 限制的类/接口>

<T extends XXX>

  限制类型参数为数字的示例：
  其中max()方法的参数类型必须是Number类的子类。

public class GenericBorderClass {

    public static <T extends Number> T max(T t1,T t2){
        return t1.doubleValue() > t2.doubleValue() ? t1 : t2;
    }

    public static void main(String[] args) {
        System.out.println(max(5,2.2));
    }
}

  类型边界可以设置多个，各边界使用&分隔。除第一个可以是类或接口，其余只能是接口。语法形式如下： 

<T extends B1 & B2>

  多个边界类型参数的方法示例： 

public class GenericMultiBorderClass {

    static class A{}    //待继承类
    interface B{};  //待继承接口
    //多个边界类型参数的泛型方法
    public static <T extends A & B> String getClassType(T t){
        return t.getClass().toGenericString();
    }
    //为泛型方法准备的参数类型
    static class C extends A implements B{}

    public static void main(String[] args) {
        System.out.println(getClassType(new C()));
    }
}

//输出
//static class com.cocoas.base.generic.GenericMultiBorderClass$C

类型通配符

类型通配符一般使用?代替具体的类型参数，表示的是未知类型的泛型。

例如List<?>可以表示List<String>、List<Integer>等类型的列表。

  1. 上界通配符

    上界通配符用于限定类型参数的类型范围为指定类型或其子类类型。

  语法形式： 

<? extends XXX>

  上界通配符使用示例： 

public class GenericGeneralTypeClass {

    //上界通配符：限定List的元素为Number或其子类Integer、Double等
    public static double sum(List<? extends Number> numbers){
        double sum = 0;
        for(Number num : numbers){
            sum += num.doubleValue();
        }
        return sum;
    }

    public static void main(String[] args) {
        System.out.println(sum(Arrays.asList(1,2.2,3.3)));
    }

}

  注意事项：
    无法往 上界通配符限定的列表中添加任何对象，除了null。

  1. 下上界通配符

    下界通配符用于限定类型参数为指定类型或其父类类型。

  语法形式： 

<? super XXX>

  下界通配符使用示例： 

public class GenericGeneralTypeClass {

    //下界通配符
    public static void appendList(List<? super Number> numbers){
        numbers.add(new Integer(123));
        numbers.add(new Double(1.23));
    }

}

  为什么上面定义的下界通配符可以添加Number的子类？因为Number的子类可以通过向上转型而实例化父类，从而可以正常添加。
  注意事项：
    列表中使用下界通配符限定后，列表将无法进行迭代。因为我们无法知晓使用哪个接口去迭代列表，即无法保证列表中元素都拥有哪个接口。

  1. 无界通配符

    未知类型的通配符称之为无界通配符。即单个问号。

  语法形式： 

<?>

  使用场景：

  • 编写一个使用Object类提供的功能来实现的方法时。

  • 在代码中使用泛型类里不依赖类型参数的方法时。
    例如：使用List中的size方法、clear方法等

  无界通配符使用示例： 

public class GenericGeneralTypeClass {

    //无界通配符
    public static void printList(List<?> numbers){
        for(int i = 0;i<numbers.size();i++){
            System.out.println(numbers.get(i).toString());
        }
    }

    public static void main(String[] args) {
        List<Integer> integerList = Arrays.asList(1,2,3);
        List<String> stringList = Arrays.asList("integer","String");
        printList(integerList); //打印Integer列表
        printList(stringList);  //打印String列表
    }

}

  注意事项：
    List<?>与List<Object>并不等同，List<Object>是List<?>的子类。
    无法往List<?>中添加任意对象，除了null。

  List<?>、List<Object>、List<T>的区别：
    List<?>指的是未知类型的泛型类型列表，即对于任何具体类型A，List<A>都是List<?>的子类型。List<?>可以代表List<A>，也可以代表List<B>、List<C>等，所以List<?>也可以代表List<Object>，即List<Object>是List<?>的子类型。
    List<Object> 只是代表列表中的元素对象可以是Object或其子类Integer、String等，List<Integer>、List<String>并不是List<Object>的子类型。不同于List<?>，我们可以向List<Object>中插入Object或其子类Integer、String等的对象。
    List<T>中的T是某一个数据类型，表示List中的元素类型都是T类型，只是在运行时才能确定T的具体类型。因为List<T>中的数据类型是固定的，所以支持添加、删除等操作。

泛型约束

  1. 泛型类型的类型参数不能是原始类型（Primitive Type）
    Java原始类型：byte、short、char、int、long、float、double、boolean 
    public class Pair<K,V> {}
Pair<int, char> pair = new Pair<>(); //编译错误
  2. 不能创建类型参数的实例 
    public static <E> void append(List<E> list){
    E element = new E(); //编译错误
    list.add(element);
}
  3. 不能将静态成员的类型声明为类型参数 
    public class Pair<K,V> {
    public static K KEY; //编译错误
}
  4. 泛型类型参数无法使用类型转换或instanceOf 
    public static <E> void append(List<E> list){
    if(list instanceof ArrayList<String>){} //编译错误
    List<Integer> integers = new ArrayList<>();
    List<String> strings = (List<String>) integers; //编译错误
}
  5. 无法创建类型参数的数组 
    List<String>[] arrayOfLists = new List<String>[2]; //编译错误
  6. 不能创建、catch、throw参数化类型对象 
    class MathException<T> extends Exception{} //编译错误
class MathException<T> extends Throwable{} //编译错误

public static <T extends Exception,J> void execute(List<J> jobs){
    try{
        for(J j : jobs){}
    }catch (T t){   //编译错误
    }
}
  7. 不能重载形参类型会被擦除成相同原始类型的方法 
    public void print(Set<String> strSet){}  //编译错误
public void print(Set<Integer> intSet){}

  突破泛型约束
  虽然泛型限定了集合中数据的类型，但由于泛型信息会在编译期间被擦除，所以在运行期间，我们就可以利用反射向集合中插入任意类型的数据。 

ArrayList<Integer> arrayInteger=new ArrayList<>();
arrayInteger.add(123);
//利用反射向arrayInteger中插入字符串
arrayInteger.getClass().getMethod("add",Object.class).invoke(arrayInteger,"abc");
for(int i = 0;i<arrayInteger.size();i++){
    System.out.println(arrayInteger.get(i));
}
//输出
123
abc

  但是并不推荐使用这种方式操作泛型类型，因为这违背了泛型的初衷：减少强制类型转换以确保类型安全。

泛型命名

  E — Element
  表示元素，主要由Java集合(Collections)框架使用。
  K — Key
  主要用于表示映射中的键的参数类型。
  V — Value
  主要用于表示映射中的值的参数类型。
  N — Number
  主要用于表示数字。
  T — Type
   主要用于表示第一类通用型参数。
  S、U、V等
   主要用于表示第二个、第三个、第四个等参数
