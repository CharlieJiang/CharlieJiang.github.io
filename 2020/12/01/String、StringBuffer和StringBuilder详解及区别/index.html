String、StringBuffer、StringBuilder和StringJoiner详解及区别

发表于 2020-12-01 分类于 Java基础 阅读次数：

String

String表示字符串对象，用于创建和操作字符串。

  字符串存储方式
  String中使用字符数组char[]来存储字符串,String中许多方法的实现都是直接对字符数组进行操作的，包括legnth()、charAt()等方法。

  字符串不可变性
  String是通过char[]实现的，而且char[]是使用final修饰的，即char[]是无法修改的。所以导致String也是不可修改的。String中针对字符窜的裁剪、拼接操作都会生成新的字符串对象去存储处理后的字符串。

public final class String
    implements java.io.Serializable, Comparable<String>, CharSequence {
    /** The value is used for character storage. */
    private final char value[];

  字符串常量池（String Constant Pool）
    字符串是Java中最基础的数据类型，大量频繁地创建字符串，极大的影响程序的性能。
    Java虚拟机为了提高程序性能和减少内存开销，在实例化字符串对象时，为字符串开辟了一个字符串常量池，类似于字符串缓存。
    作用：缓存String对象，以达到重复利用的目的。
    工作原理：当需要使用字符串字面量创建字符串对象时，会首先去字符串常量池中查询是否存在与此字符串字面量相同的字符串对象，如果存在，则返回此字符串对象的引用；否则，创建新的字符串对象放入字符串常量池中并返回其引用。
    字符串字面量：字符串字面量就是两个双引号之间的字符序列。如”cocoas”、“Java”等。
    存储位置：
      JDK1.7之前：JVM运行时数据区的方法区中
      JDK1.7之后：JVM运行时数据区的堆中
    字符串对象重复使用示例： 

public static void main(String[] args) {
    String str1 = "cocoas";
    String str2 = "cocoas";
    System.out.println(str1 == str2);
}

//返回 true

  注意事项

  • 两种创建字符串对象的区别
    通过字符串字面量创建字符串对象和通过new String()对象创建字符串对象是不同的，后者是不会去查询字符串常量池中的缓存，而是直接在堆中创建新的字符串对象。
      new String()创建字符串使用示例： 
    public static void main(String[] args) {
    String str2 = "cocoas";
    String str3 = new String("cocoas");
    System.out.println(str2 == str3);
    String str4 = new String("cocoas");
    System.out.println(str4 == str3);
}

//返回 false false
  • 字符串字面量和字符串对象的拼接方式是不同的
    多个字符串字面量的拼接和多个字符串对象的拼接是不同的，前者会从字符串常量池查询拼接后的字符串字面量是否存在，不存在则创建，然后返回引用；后者会创建StringBuilder对象，然后使用StringBuilder.appen()方法对字符串进行拼接，最后通过StringBuilder.toString()方法返回拼接后的字符串对象。因StringBuilder.toString()是通过new String()创建字符串对象返回的，所以这两种方式返回的结果并不相同。
      两种字符串拼接方式示例： 
    public static void main(String[] args) {
    String str1 = "cocoas";
    String str2 = "cocoas";
    String str5 = "cocoas" + "cocoas";
    // 相当于：new StringBuilder().append(str1).append(str2).toString();
    String str6 = str1 + str2;
    System.out.println(str5 == str6);
}

//返回 false

StringBuffer

StringBuffer是一个线程安全的、可变的字符序列。

  String对象本身是无法修改的，当我们需要对字符串进行修改、拼接时，可以使用Java 提供的StringBuffer和StringBuilder。
  工作原理
    StringBuffer通过继承抽象类AbstractStringBuilder实现字符串的存储，AbstractStringBuilder存储字符串的方式与String一样，也是通过字符数组char[]进行存储。与String不同的是，AbstractStringBuilder中的char[]是没有final修饰符的，即AbstractStringBuilder中的字符串是可以修改的。AbstractStringBuilder也正是通过对字符数组进行扩容或缩容来实现对字符串的修改。
  AbstractStringBuilder使用字符数组存储字符串：

abstract class AbstractStringBuilder implements Appendable, CharSequence {
    /**
     * The value is used for character storage.
     */
    char[] value;

  AbstractStringBuilder对字符数组进行扩容： 

public AbstractStringBuilder append(char c) {

private void ensureCapacityInternal(int minimumCapacity) {
    // overflow-conscious code
    if (minimumCapacity - value.length > 0) {
        value = Arrays.copyOf(value,
                newCapacity(minimumCapacity));//字符数组扩容
    }

public AbstractStringBuilder deleteCharAt(int index) {

  StringBuffer是不可继承的：StringBuffer类是有final修饰符的，即无法继承的。

  StringBuffer是线程安全的
    StringBuffer中大部分方法都是带有synchronized修饰符的，有些虽然没有synchronized修饰符，但是可通过调用重载方法或是StringBuffer的其它方法实现同步。所以，StringBuffer是线程安全的。
  StringBuffer中带synchronized修饰符的方法示例： 

public synchronized StringBuffer append(String str) {
    toStringCache = null;
    super.append(str);
    return this;
}

  StringBuffer中通过调用重载方法实现同步的方法示例： 

public  StringBuffer insert(int offset, boolean b) {
    // Note, synchronization achieved via invocation of StringBuffer insert(int, String)
    // after conversion of b to String by super class method
    // Ditto for toStringCache clearing
    super.insert(offset, b);
    return this;
}

  StringBuffer中通过调用其他StringBuffer方法实现同步的方法示例： 

public int indexOf(String str) {
    // Note, synchronization achieved via invocations of other StringBuffer methods
    return super.indexOf(str);
}

StringBuilder

StringBuilder是一个非线程安全的、可变的字符序列。

  与StringBuffer一样，StringBuilder也是继承自AbstractStringBuilder。StringBuilder与StringBuffer功能相同，区别在于前者是非线程安全的，而后者是线程安全的。
  使用场景区别

  • 单线程环境，StringBuilder一般是略优于StringBuffer的，推荐使用StringBuilder。
  • 多线程环境，StringBuilder是非线程安全的，所以应该使用StringBuffer。

StringJoiner

StringJoiner是JDK 1.8新加入的字符串拼接工具类。

  StringJoiner是一个专门用来构造以指定分隔符分隔的字符序列，并且可选设定以指定前缀开始及以指定后缀结束。String.join()方法就是通过StringJoiner对象实现的。
  虽然StringBuffer、StringBuilder等都可以通过一定方式实现StringJoiner的功能，但StringJoiner提供了更简单的实现方式，因而更方便使用。

  StringJoiner实现原理
    StringJoiner内部是通过StringBuilder来实现的。

  • StringJoiner通过prepareBuilder()方法构造一个带前缀的StringBuilder对象： 
    private StringBuilder prepareBuilder() {
    if (value != null) {
        value.append(delimiter);
    } else {
        value = new StringBuilder().append(prefix);
    }
    return value;
}
  • 在每次调用add()拼接字符串前，都会调用prepareBuilder()方法获取StringBuilder对象并预先添加分隔符delimiter，然后使用StringBuilder的append方法拼接字符串： 
    public StringJoiner add(CharSequence newElement) {
    prepareBuilder().append(newElement);
    return this;
}
      StringJoiner使用
  • StringJoiner通过构造器传入分隔符和前缀后缀： 
    public StringJoiner(CharSequence delimiter) {
    this(delimiter, "", "");
}
public StringJoiner(CharSequence delimiter,
                    CharSequence prefix,
                    CharSequence suffix) {
    Objects.requireNonNull(prefix, "The prefix must not be null");
    Objects.requireNonNull(delimiter, "The delimiter must not be null");
    Objects.requireNonNull(suffix, "The suffix must not be null");
    // make defensive copies of arguments
    this.prefix = prefix.toString();
    this.delimiter = delimiter.toString();
    this.suffix = suffix.toString();
    this.emptyValue = this.prefix + this.suffix;
}
  • StringJoiner通过add()方法拼接字符串： 
    public StringJoiner add(CharSequence newElement) {
    prepareBuilder().append(newElement);
    return this;
}
  • StringJoiner通过toString()方法返回最终的字符串：先调用StringBuilder.append()添加后缀，然后调用StringBuilder.toString()获取最终的拼接结果。 
    public String toString() {
    if (value == null) {
        return emptyValue;
    } else {
        if (suffix.equals("")) {
            return value.toString();
        } else {
            int initialLength = value.length();
            String result = value.append(suffix).toString();
            // reset value to pre-append initialLength
            value.setLength(initialLength);
            return result;
        }
    }
}
