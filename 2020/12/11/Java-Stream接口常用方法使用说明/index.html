Java Stream接口常用方法使用说明

发表于 2020-12-11 更新于 2020-12-30 分类于 Java基础 ， Java Stream 阅读次数：

Stream方法使用非常依赖于函数式接口。Stream的众多方法中使用了JDK预设的一些函数式接口，包括Function<T,R>、Predicate、Supplier等。由于lambda表达式可以表示函数式接口的实现，而且实现比较简洁，所以通常都是Stream方法与lambda表达式一起使用。

allMatch()

  • 方法定义： 
    boolean allMatch(Predicate<? super T> predicate)
  • 方法作用：判断Stream中所有元素是否均满足predicate函数指定的条件
  • 使用示例：
    List<String> list = Arrays.asList("one","two","three");
boolean allMatch = list.stream().allMatch(s -> s.length() > 3);
System.out.println(allMatch); // 输出 false

anyMatch()

  • 方法定义： 
    boolean anyMatch(Predicate<? super T> predicate);
  • 方法作用：判断Stream中是否存在元素满足predicate函数指定的条件，存在则停止遍历返回，否则继续遍历直到所有元素完成条件判断。
  • 使用示例：
    List<String> list = Arrays.asList("one","two","three");
boolean anyMatch= list.stream().anyMatch(s -> s.length() > 3);
System.out.println(anyMatch); // 输出 true

collect(collector)

  • 方法定义： 
    <R, A> R collect(Collector<? super T, A, R> collector)
  • 方法作用：将流元素收集到collector指定的收集器中
  • 使用示例：
    Stream<String> stream = Stream.of("one","two","three");
List<String> stringList = stream.collect(Collectors.toList());

collect(supplier,accumulator,combiner)

  • 方法定义： 
    <R> R collect(Supplier<R> supplier,
                 BiConsumer<R, ? super T> accumulator,
                 BiConsumer<R, R> combiner);
  • 方法作用：对流元素执行可变规约操作，产生一个可变结果容器。
    其实该方法作用与collect(collector)相同，Collector内部就是通过封装supplier,accumulator,combiner这三个参数来实现的，以简化代码书写。
    • supplier：一个构造器，负责创建一个容器对象，用来收集数据，如构造一个ArrayList容器就是ArrayList::new
    • accumulator：累加器，负责向容器中逐一添加数据，如向ArrayList中添加数据就是ArrayList::add
    • combiner：合并器，负责合并多个中间结果，即合并并行计算时多个线程执行的结果，也就是集合间合并，如合并ArrayList集合就是ArrayList::addAll
  • 使用示例：
    Stream<String> stream = Stream.of("one","two","three");
List<String> strings = stream
                .collect(ArrayList::new,
                        ArrayList::add,
                        (list1,list2) -> list1.addAll(list2));

concat()

  • 方法定义：
    public static <T> Stream<T> concat(Stream<? extends T> a, Stream<? extends T> b)
  • 方法作用：合并两个同类型的Stream
  • 使用示例：
    Stream<String> a = Stream.of("I'm","in","love");
Stream<String> b = Stream.of("with","cocoas");
Stream<String> ab = Stream.concat(a,b);
ab.forEach(s -> System.out.print(s + " "));
// 输出
I'm in love with cocoas

    count()

  • 方法定义：
    long count()
  • 方法作用：返回Stream中元素个数
  • 使用示例：
    List<String> list = Arrays.asList("one","two","three");
long count = list.stream().count();
System.out.println(count); // 输出 3

distinct()

  • 方法定义：
    Stream<T> distinct()
  • 方法作用：根据Object.equal()方法去除Stream中的重复元素。
  • 使用示例：
    List<String> list = Arrays.asList("one","two","three","one","two");
list.stream().distinct().forEach(s -> System.out.println(s));
// 输出
one
two
three

    empty()

  • 方法定义：
    public static<T> Stream<T> empty()
  • 方法作用：返回一个空的Stream。
  • 使用示例：
    long count = Stream.empty().count();
System.out.println(count);
// 输出
0

filter()

  • 方法定义：
    Stream<T> filter(Predicate<? super T> predicate)
  • 方法作用：过滤掉Stream中不满足predicate函数指定的条件的元素。
  • 使用示例：
    List<String> list = Arrays.asList("one","two","three");
list.stream()
        .filter(s -> s.length() > 3)
        .forEach(s -> System.out.println(s));
// 输出
three

findAny()

  • 方法定义：
    Optional<T> findAny()
  • 方法作用：找到Stream中任意一个元素，并将元素封装成Optional对象后返回。Optional是对泛型对象做空指针处理的一个类。
    • 在串行流中，findAny返回的就是流中的第一个元素，此时findAny等同于findFirst。
    • 在并行流中，findAny返回的是最快处理完的那个线程的数据。
  • 使用示例：
    List<String> list = Arrays.asList("one","two","three");
Optional<String> anyStr = list.stream().findAny();
if(anyStr.isPresent()){
    System.out.println(anyStr.get());
}
// 输出
one

findFirst()

  • 方法定义
    Optional<T> findFirst()
  • 方法作用：找到Stream中第一个元素，并将元素封装成Optional对象后返回。
  • 使用示例：
    List<String> list = Arrays.asList("one","two","three");
Optional<String> firstStr= list.stream().findFirst();
if(firstStr.isPresent()){
    System.out.println(firstStr.get());
}
// 输出
one

flatMap()

  • 方法定义
    <R> Stream<R> flatMap(Function<? super T, ? extends Stream<? extends R>> mapper)
  • 方法作用：对Stream中每个元素执行mapper函数指定的操作，每个元素执行完会产生一个新的Stream叫做Mapped Stream，然后以所有Mapped Stream中的元素组成一个新的Stream返回。简单来说，就是将原Stream中的所有元素都“摊平”后组成新的Stream，转换前后元素的个数和类型都可能会改变。
  • 类似方法：flatMapToDouble()、flatMapToInt()、flatMapToLong()
  • 使用示例
    // 将List<String>类型的流转换为String类型的流
Stream<List<String>> listStream = Stream.of(
                Arrays.asList("I'm","in","love"),
                Arrays.asList("with","cocoas"));
        Stream<String> strStream = listStream
                .flatMap(strList -> strList.stream());
        strStream.forEach(s -> System.out.print(s + " "));
// 输出
I'm in love with cocoas

forEach()

  • 方法定义
    void forEach(Consumer<? super T> action)
  • 方法作用：对Stream中每个元素执行action函数指定的操作，执行完并不会返回任何结果，常用作元素遍历。
  • 类似方法：forEachOrdered()
  • 使用示例
    List<String> list = Arrays.asList("one","two","three");
list.stream().forEach(s -> System.out.println(s));
// 输出
one
two
three

iterate()

  • 方法定义
    public static<T> Stream<T> iterate(final T seed, final UnaryOperator<T> f)
  • 方法作用：返回一个无限个数元素的有序Stream，其中第一个元素是seed，后面每一个元素都是前一个元素通过应用函数f得到的，即返回的Stream中的元素依次是seed、f(seed)、f(f(seed))等。
  • 使用示例
    Stream<Integer> intStream = Stream.iterate(1,i -> i * 2);
intStream.limit(10).forEach(i -> System.out.print(i + " "));
// 输出
1 2 4 8 16 32 64 128 256 512

limit()

  • 方法定义
    Stream<T> limit(long maxSize)
  • 方法作用：返回由原Stream中前maxSize个元素组成的新Stream,如果原Stream中元素个数小于maxSize，则返回由原Stream中所有元素组成的新Stream。
  • 使用示例
    List<String> list = Arrays.asList("one","two","three");
list.stream().limit(2).forEach(s -> System.out.println(s));
// 输出
one
two

map()

  • 方法定义
    <R> Stream<R> map(Function<? super T, ? extends R> mapper)
  • 方法作用：对Stream中所有元素执行mapper函数所指定的操作，返回以mapper操作的结果组成的新Stream。map()相当于对元素进行转换处理，转换后元素个数不变，但元素类型可能会改变。
  • 相似方法：mapToDouble()、mapToInt()、mapToLong()
  • 使用示例
    List<String> list = Arrays.asList("one","two","three");
// 元素类型不变
list.stream()
        .map(s -> s.toUpperCase())
        .forEach(i -> System.out.print(i + " "));
// 输出 ONE TWO THREE
// 元素类型改变
list.stream()
        .map(s -> s.length())
        .forEach(i -> System.out.print(i + " "));
// 输出 3 3 5

max()

  • 方法定义
    Optional<T> max(Comparator<? super T> comparator)
  • 方法作用：根据提供的比较函数comparator对Stream中元素进行比较，返回Stream中最大的元素。
  • 相似方法：min()
  • 使用示例
    List<String> list = Arrays.asList("one","two","three");
Optional max = list.stream()
        .max((s1,s2) -> s1.length() > s2.length() ? 1 : -1); //根据长度比较字符串大小
if(max.isPresent()){
    System.out.println(max.get());
}
// 输出
three

noneMatch()

  • 方法定义
    boolean noneMatch(Predicate<? super T> predicate)
  • 方法作用：判断Stream中是否无元素满足predicate函数指定的条件。该方法找到符合条件的元素就会返回false，所以可能不会遍历完Stream中所有元素。
  • 使用示例
    List<String> list = Arrays.asList("one","two","three");
boolean noneMatchResult = list.stream()
        .noneMatch(s -> s.endsWith("coco"));
System.out.println(noneMatchResult);
// 输出
true

of()

  • 方法定义
    public static<T> Stream<T> of(T t)
  • 方法作用：创建一个只包含一个元素t的Stream
  • 相似方法：of(T… values)
  • 使用示例
    Stream<String> singleItemStream = Stream.of("cocoas");
singleItemStream.forEach(s -> System.out.println(s));
// 输出
cocoas

peek()

  • 方法定义
    Stream<T> peek(Consumer<? super T> action)
  • 方法作用：通过action函数指定的操作遍历元素，类似于forEach()方法，区别在于peek()方法会返回与原Stream中元素完全相同的新Stream。
    peek()方法主要是为了支持debug的，在debug时使用peek()方法，既可以查看某些流操作后元素的情况，也不会影响后续的流操作执行。
  • 使用示例
    List<String> list = Arrays.asList("one","two","three");
list.stream()
        .map(s -> s.toUpperCase()) // 字符串转大写
        .peek(s -> System.out.print(s + " ")) // 查看map()操作的结果
        .forEach(s -> System.out.print(s + " ")); // 查看peek()操作后是否会影响map()操作的结果
// 输出
ONE ONE TWO TWO THREE THREE

reduce()

  • 方法定义
    Optional<T> reduce(BinaryOperator<T> accumulator)
T reduce(T identity, BinaryOperator<T> accumulator)
<U> U reduce(U identity,
             BiFunction<U, ? super T, U> accumulator,
             BinaryOperator<U> combiner)
  • 方法作用：从Stream的所有元素中取出一个值，例如取元素最大最小值，或者生成一个新值，例如取元素个数、元素总和、元素拼接等，返回一个包装此值得Optional对象，Optional类中对null做了处理。
  • 使用示例
    // 取出长度最大的元素
List<String> list = Arrays.asList("one","two","three");
Optional<String> reduceOptional = list.stream()
        .reduce(BinaryOperator.maxBy((s1,s2) -> s1.length() > s2.length() ? 1 : -1));
if(reduceOptional.isPresent()){
    System.out.println(reduceOptional.get());
}
// 输出
three
// 计算所有元素的长度总和
int sumLength = list.stream()
        .reduce(0, // 初始值
                (sum,s) -> sum += s.length(), // 累加器
                (sum1,sum2) -> sum1 + sum2); // 部分和拼接：并行执行时才会用到
System.out.println(sumLength);
// 输出
11

skip()

  • 方法定义
    Stream<T> skip(long n)
  • 方法作用：丢弃掉原Stream中前n个元素，返回由剩余元素组成的Stream，如果原Stream元素个数不足n，则返回空的Stream。
  • 使用示例
    List<String> list = Arrays.asList("one","two","three");
list.stream().skip(2)
        .forEach(s -> System.out.println(s));
// 输出
three

sorted()

  • 方法定义
    Stream<T> sorted()
Stream<T> sorted(Comparator<? super T> comparator);
  • 方法作用：对Stream中元素进行排序。
    • 无参数时，会根据自然顺序进行排序。如果元素不可排序，则会在执行最终操作时抛出类型转换异常。
    • 有参数时，会根据指定的比较函数comparator进行排序。
  • 使用示例
    List<String> list = Arrays.asList("one","two","three");
        list.stream().sorted()
                .forEach(s -> System.out.println(s));
// 输出
one
three
two
