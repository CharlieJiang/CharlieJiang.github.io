Java Stream 基础

发表于 2020-12-08 更新于 2020-12-10 分类于 Java基础 阅读次数：

什么是函数式编程

  简单说，函数式编程是一种编程规范，主要思想是把运算过程尽量写成一系列嵌套的函数调用。举例来说，现在有这么一个数学表达式： 

(1 + 2) * 3 -4

  传统的过程式编程可能这样写： 

int a = 1 + 2;
int b = a * 3;
int c = b -4;

  函数式编程要求使用函数，我们可以把运算过程定义成不同的函数，然后写成下面这样： 

int result = subtract(multiply(add(1,2),3), 4);

什么是Stream

  Stream是指支持顺序操作和并行聚合操作的元素序列（A sequence of elements supporting sequential and parallel aggregate operations）。其中的元素序列来自于数据源。
  Stream并不是某种数据结构，它只是数据源的一种视图。这里的数据源可以是数组、集合、I/O Channel、产生器generator等。
  Stream是Java函数式编程的主角。

如何创建Stream

  因为Stream只是数据源的一种视图，如果我们想要得到一个Stream，通常不会手动去创建，而是调用对应的工具方法。例如：

  1. 调用Collections.stream()或Collections.parallelStream()方法
    Collections.stream()示例：使用List数据源创建Stream
    // 使用List创建Stream
List<String> list = Arrays.asList("one","two","three");
list.stream().forEach(s -> System.out.println(s));
      Collections.parallelStream()：使用Set数据源创建并行Stream
    // 使用Set创建Stream
Set<Integer> set = new HashSet<>();
set.add(1);set.add(2);set.add(3);
int max = set.parallelStream().max((i,j) -> i > j ? 1 : -1).get();
System.out.println(max);
  2. 调用Arrays.stream(T[] array)方法
    使用数组创建Stream示例：
    // 使用数组创建Stream
String array[] = {"one","two","three"};
Arrays.stream(array)
        .filter(s -> s.startsWith("t"))
        .forEach(s -> System.out.println(s));
  3. 调用Stream.of(T t)/Stream.of(T… values)方法
    使用Stream.of(T t)创建只含单个元素的Stream示例：
    Stream<String> singleStream = Stream.of("cocoas");
singleStream.forEach(s -> System.out.println(s));
      使用Stream.of(T… t)创建Stream示例： 
    // 方式一：多个同类型数据
Stream<String> multiStream = Stream.of("one","two","three");
multiStream.forEach(s -> System.out.println(s));
// 方式二：数组
String array[] = {"one","two","three"};
Stream<String> multiStream2 = Stream.of(array);
multiStream2.forEach(s -> System.out.println(s));

Stream与Collections的区别

  虽然Stream一般都是通过集合的Collection.stream()方法生成，但Stream和Collection还是有以下不同：

  • 无存储：Stream不是一种数据类型，只是数据源的一种视图，并不会存储数据。
  • 函数式编程：对Stream的任何操作并不会改变其背后的数据源，比如对Stream执行过滤操作并不会删除被过滤的元素，而是生成不包含被过滤元素的新Stream。
  • 惰式执行：Stream上的操作并不会立即执行，只有当用户调用结束操作时才会执行。
  • 可消费性：Stream只能被”消费“一次，一旦遍历过后就会失效，类似容器的迭代器，想要再次遍历必须重新生成。

常见Stream接口

接口名称 接口说明
BaseStream<T,S extends BaseStream<T,S>> 流的基本接口，定义了流相关的一些基本操作。BaseStream也是其他Stream接口的父接口。
IntStream 继承自BaseStream接口，专门用于处理整形数据的Stream接口。
LongStream 继承自BaseStream接口，专门用于处理长整形数据的Stream接口。
DoubleStream 继承自BaseStream接口，专门用于处理浮点型数据的Stream接口。
Stream 继承自BaseStream接口，用于处理除以上数据类型外的其他类型数据的Stream接口。如数组、JavaBean等。

Stream<T>与JavaBean使用示例：

public class StreamClass {

    public static void main(String[] args) {
        Stream<Student> studentStream = Stream.of(new Student("cocoas"),new Student("小芳"));
        studentStream.forEach(stu -> System.out.println(stu.name));
    }

    static class Student{
        public Student(String name){
            this.name = name;
        }
        String name = "";
    }
}

// 输出
cocoas
小芳

Stream的两种操作：中间操作和结束操作

  中间操作
    中间操作都是惰式执行，调用中间操作会生成一个标记该操作的新Stream。即方法返回值为Stream类型的基本都是中间操作。
  结束操作
    结束操作才会触发真正的计算，计算发生时会把所有中间操作以pipeline的方式执行，这样可以减少迭代次数。计算完成后Stream就会失效。
  Stream中常用的中间操作和结果操作方法如下：

操作类型 操作方法
中间操作 concat() distinct() filter() flatMap() limit() map() peek() skip() sorted() parallel() sequential() unordered()
结束操作 allMatch() anyMatch() collect() count() findAny() findFirst() forEach() forEachOrdered() max() min() noneMatch() reduce() toArray()

Stream常用方法

  Stream方法使用非常依赖于函数式接口。Stream的众多方法中使用了JDK预设的一些函数式接口，包括Function<T,R>、Predicate、Supplier等。由于lambda表达式可以表示函数式接口的实现，而且实现比较简洁，所以通常都是Stream方法与lambda表达式一起使用。

  • distinct()
    方法定义： 
    Stream<T> distinct()
    方法作用：根据Object.equal()方法去除Stream中的重复元素。
    使用示例：
    List<String> list = Arrays.asList("one","two","three","one","two");
list.stream().distinct().forEach(s -> System.out.println(s));
// 输出
one
two
three
  • filter()
    方法定义：
    Stream<T> filter(Predicate<? super T> predicate)
    方法作用：过滤掉Stream中不满足predicate函数指定的条件的元素。
    使用示例：
    List<String> list = Arrays.asList("one","two","three");
list.stream()
        .filter(s -> s.length() > 3)
        .forEach(s -> System.out.println(s));
// 输出
three
  • flatMap()
    方法定义：
    <R> Stream<R> flatMap(Function<? super T, ? extends Stream<? extends R>> mapper)
    方法作用：对Stream中每个元素执行mapper函数指定的操作，每个元素执行完会产生一个新的Stream叫做Mapped Stream，然后以所有Mapped Stream中的元素组成一个新的Stream返回。简单来说，就是将原Stream中的所有元素都“摊平”后组成新的Stream，转换前后元素的个数和类型都可能会改变。
    类似方法：flatMapToDouble()、flatMapToInt()、flatMapToLong()
    使用示例：
    // 将List<String>类型的流转换为String类型的流
Stream<List<String>> listStream = Stream.of(Arrays.asList("I'm","in","love"),Arrays.asList("with","cocoas"));
Stream<String> strStream = listStream.flatMap(strList -> strList.stream());
strStream.forEach(s -> System.out.print(s + " "));
// 输出
I'm in love with cocoas
  • forEach()
    方法定义：
    void forEach(Consumer<? super T> action)
    方法作用：对Stream中每个元素执行action函数指定的操作，执行完并不会返回任何结果，常用作元素遍历。
    相似方法：forEachOrdered()
    使用示例：
    List<String> list = Arrays.asList("one","two","three");
list.stream().forEach(s -> System.out.println(s));
// 输出
one
two
three
three
  • map()
    方法定义：
    <R> Stream<R> map(Function<? super T, ? extends R> mapper)
    方法作用：对Stream中所有元素执行mapper函数所指定的操作，返回以mapper操作的结果组成的新Stream。map()相当于对元素进行转换处理，转换后元素个数不变，但元素类型可能会改变。
    相似方法：mapToDouble()、mapToInt()、mapToLong()
    使用示例：
    List<String> list = Arrays.asList("one","two","three");
// 元素类型不变
list.stream()
        .map(s -> s.toUpperCase())
        .forEach(i -> System.out.print(i + " "));
// 输出 ONE TWO THREE
// 元素类型改变
list.stream()
        .map(s -> s.length())
        .forEach(i -> System.out.print(i + " "));
// 输出 3 3 5

Stream的归约操作

  规约操作（Reduction Operation）又称作折叠操作，是通过某个连接操作将所有元素汇总成一个汇总结果的过程。元素求和、求最大值最小值、求元素总数、将所有元素转换成一个列表或集合等操作都属于规约操作。
  Stream类库中有两个通用的规约操作reduce()和collect()，也有一些为了简化书写而设计的专用规约操作，包括sum()、max()、min()、count()等。

  • reduce()

reduce操作可以实现从Stream的所有元素中产生一个值，这个值可以是Stream中原有的元素，也可以是操作后产生的新值。我们常用的sum()、max()、min()、count()等方法都是reduce操作。
方法定义：reduce拥有三种重载方法

Optional<T> reduce(BinaryOperator<T> accumulator) // 方法1
T reduce(T identity, BinaryOperator<T> accumulator) // 方法2
<U> U reduce(U identity, // 方法3
             BiFunction<U, ? super T, U> accumulator,
             BinaryOperator<U> combiner)

  方法1的参数只有一个累加器函数，只需要定义两个元素间的连接操作，返回的是一个封装了所有元素连接结果的Optional类型值。Optional类是一个值容器，其内部对null值做了封装处理，使用它可以避免null值的麻烦。通过Optional.isPresent()可以判断Optional中是否有值，通过Optional.get()可以获取Optional中的值。方法1使用示例： 

// 查找最长字符串
List<String> list = Arrays.asList("one","two","three");
Optional<String> reduceOptional = list.stream()
        .reduce((s1,s2) -> s1.length() > s2.length() ? s1 : s2);
if(reduceOptional.isPresent()){
    System.out.println(reduceOptional.get());
}
// 输出
three

  上述示例中，lambda表达式中的参数s1是表达式执行结果的缓存，也就是表达式每次执行的结果都会作为下次执行时的第一个入参；参数s2是就是流中的每个元素。表达式第一次执行时，参数s1就是流中的第一个元素。
  方法2比方法1多了一个参数identity，identity是连接操作的初始值。即lambda表达式第一次执行的时候，s1不再是流中的第一个元素，而是identity。方法2使用示例： 

// 查找最长字符串
List<String> list = Arrays.asList("one","two","three");
String reduceStr  = list.stream()
        .reduce("eleven",(s1,s2) -> s1.length() > s2.length() ? s1 : s2);
System.out.println(reduceStr);
// 输出
eleven

  方法3比方法2多了一个参数combiner，combiner函数指定了并行执行时多个线程连接结果的合并方式。方法1和方法2只能返回与元素类型相同的数据类型（方法1会对返回的数据进行null封装），但方法3的返回值类型与元素类型可以不同。利用这点，我们就能直接计算查找到的最长字符串的长度、计算所有字符串长度总和等并可将结果直接返回。方法3使用示例：

// 查找最长字符串的长度
List<String> list = Arrays.asList("one","two","three");
int maxLength = list.stream()
        .reduce(0,
                (maxLen,s) -> maxLen > s.length() ? maxLen : s.length(),
                (maxLen1,maxLen2) -> maxLen1 > maxLen2 ? maxLen1 : maxLen2);
System.out.println(maxLength);
// 输出
5
// 计算所有元素的长度总和
int sumLength = list.stream()
        .reduce(0, // 初始长度
                (sum,s) -> sum += s.length(), // 长度累加
                (sum1,sum2) -> sum1 + sum2); // 部分和拼接：并行执行时才会用到
System.out.println(sumLength);
// 输出
11
  • collect()

collect()方法使用collector函数指定的收集操作对Stream所有元素执行可变规约操作（mutable reduction operation）并返回指定数据类型为R的数据。
方法定义：collect有两种重载方法

<R, A> R collect(Collector<? super T, A, R> collector) // 方法1
<R> R collect(Supplier<R> supplier, // 方法2
              BiConsumer<R, ? super T> accumulator,
              BiConsumer<R, R> combiner);

  方法1中的collector参数其实就是对方法2中三个参数进行的封装处理，以简化书写。Collector接口部分方法如下：

public interface Collector<T, A, R> {

    Supplier<A> supplier();

    BiConsumer<A, T> accumulator();

    BinaryOperator<A> combiner();
    //其他代码
}

  收集器Collector是为collect()方法量身打造的工具接口，Collector本身就是一个可变归约操作，它能将元素收集到一个可变的结果容器中。如List、Set、Map等。Collector内部就是对supplier、accumulator和combiner这三个参数的简单封装。
    supplier：supplier函数定义了创建可变结果容器的方式，即创建容器对象的方式，如new ArrayList<>()、new HashMap<>()等
    accumulator：accumulator是一个将元素折叠进可变结果容器的函数，即定义容器元素添加的方法，如List.add()、Map.put()等
    combiner：combiner函数在并行环境才会执行，combiner定义了如何合并多个线程收集元素的结果，即如何合并多个可变结果容器中的元素，如List.addAll()、Map.putAll()等

  方法1使用示例：使用Collectors指定收集函数 

List<String> strList = Stream.of("one","two","three") // 创建一个字符串Stream
        .collect(Collectors.toList()); // 将Stream中字符串收集到List容器中
strList.stream().forEach(s -> System.out.println(s));
// 输出
one
two
three

  Collectors是基于Collector实现的一个工具类，Collectors为我们提供了许多常用的Collector，如收集到List的Collectors.toList()、收集到Map的Collectors.toMap()等，使我们可以很方便使用Stream.collect()方法。
  使用Collectors.toMap()收集元素到Map中示例： 

// 生成以字符串为key，以字符串长度为value的Map
Map<String,Integer> map = Stream.of("one","two","three")
        .collect(Collectors.toMap(Function.identity(),s -> s.length()));
System.out.println(map.get("two"));
// 输出
3

  其中Function.identity()是一个输入与输出相同的方法，是对lambda表达式t - t的简化书写。

  方法2使用示例：示例中使用了方法引用，详情请参考 方法引用 

List<String> stringList = Stream.of("one","two","three")// 创建一个字符串Stream
        .collect(ArrayList::new, // 定义创建容器方法
                ArrayList::add, // 定义容器添加元素方法
                ArrayList::addAll); // 定义多个容器结果的合并方法
stringList.stream().forEach(s -> System.out.print(s + " "));
// 输出
one two three

  collect使用详解

  • 使用collect()生成集合Collection
    将Stream转换为List、Set等集合是比较常见的操作，所以Collectors工具类中已经为我们提供了对应的收集器Collectors.toList()、Collectors.toSet()，使用示例： 
    Stream<String> stream = Stream.of("one","two","three");
// 将Stream转换为List
List<String> strlist = stream.collect(Collectors.toList()); 
// 将Stream转换为Set
Set<String> set = stream.collect(Collectors.toSet());
      当我们需要手动指定集合容器的类型时，可以使用Collectors.toCollection()方法，使用示例： 
    Stream<String> stream = Stream.of("one","two","three");
// 将Stream转换为ArrayList
ArrayList<String> arrayList = stream.collect(Collectors.toCollection(ArrayList::new)); 
// 将Stream转换为HashSet
HashSet<String> hashSet = stream.collect(Collectors.toCollection(HashSet::new));
  • 使用collect()生成Map
    虽然Stream的数据源不包括Map这种容器，但是我们还是可以从Stream中生成Map。不过在将Stream转换成Map前，首先要考虑的是如何生成Map中的Key、Value。通常三种情况下collect()的结果会是Map:
    1、使用Collectors.toMap()生成的收集器
      使用此收集器时，我们需要指定如何生成Map中的Key和Value。如将学生列表转换为<学生,GPA>组成的Map：
    // 使用Collectors.toMap()统计学生GPA
List<Student> students = new ArrayList<>();
Map<Student,Double> studentToGPA = students.stream()
        .collect(Collectors.toMap(Function.identity(), // 如何生成key
                student -> student.computeGPA())); // 如何生成value
      2、使用Collectors.partitioningBy()生成的收集器
        此收集器适合于对Stream中元素进行二分区时使用，即根据某个条件将Stream中元素分成满足条件的和不满足条件的两部分。比如将学生按性别分为男生和女生两部分：
    // 使用Collectors.partitioningBy()按性别划分学生
List<Student> students = new ArrayList<>();
Map<Boolean,List<Student>> boyGirlMap = students.stream()
        .collect(Collectors.partitioningBy(
                student -> student.getSex().equals(Student.BOY)));
      3、使用Collectors.groupingBy()生成的收集器
        此收集器适合在对元素进行分组时使用。不同于partitioningBy()只能分成两组，groupingBy()可以分为多组，而且分组方式更为灵活。类似于数据库中的group by语句，groupingBy()也是按照某个属性对数据进行分组，将属性相同的数据存放到Map中同一个key下。如下将学生按班级进行分组：
    // 使用Collectors.groupingBy()将学生按班级分组
Map<Clazz,List<Student>> studentsByClazz = students.stream()
        .collect(Collectors.groupingBy(
                Student::getClazz));

  有时候只对元素进行分组是远远不够的，我们还需要对分组后的元素进行二次处理，增强版的groupingBy()就能够满足我们这种需求。增强版的groupingBy()如下：

public static <T, K, A, D>
Collector<T, ?, Map<K, D>> groupingBy(Function<? super T, ? extends K> classifier,
                                      Collector<? super T, A, D> downstream) {
    return groupingBy(classifier, HashMap::new, downstream);
}

  增强版的groupingBy()中通过一个downstream收集器来完成我们的额外需求，允许我们对元素分组后再执行某种运算，如求和、计数、平均值、类型转换等。这种先将元素分组的收集器叫做上游收集器(upstream collector)，分组后执行其它运算的收集器叫做下游收集器（downstream Collector）。如下使用下游收集器统计每个班级的人数：

// 使用增强Collectors.groupingBy()统计每个班级的人数
Map<Clazz,Long> clazzStudentCount = students.stream()
        .collect(Collectors.groupingBy(Student::getClazz, // 上游收集器
                Collectors.counting())); // 下游收集器

  下游收集器嵌套：当下游收集器要实现的功能较为复杂时，我们需要在下游收集器中使用其它的收集器来辅助完成其功能。比如在对学生按班级进行分组后，我们希望得到每个班级中学生的姓名列表，而不是学生列表： 

// 使用增强Collectors.groupingBy()统计每个班级的人数
Map<Clazz,Long> clazzStudentCount = students.stream()
        .collect(Collectors.groupingBy(Student::getClazz, // 上游收集器
                Collectors.counting())); // 下游收集器
  • 使用collect()实现字符串拼接
    通过Collectors.joining()生成的收集器，可以实现三种常见的字符串拼接：直接拼接、带分隔符拼接、带分隔符和前后缀拼接。Collectors.joining()通过三个重载方法分别实现这三种拼接方式。joining的三种重载方法如下：
    public static Collector<CharSequence, ?, String> joining() {
    return new CollectorImpl<CharSequence, StringBuilder, String>(
            StringBuilder::new, StringBuilder::append,
            (r1, r2) -> { r1.append(r2); return r1; },
            StringBuilder::toString, CH_NOID);
}
public static Collector<CharSequence, ?, String> joining(CharSequence delimiter) {
    return joining(delimiter, "", "");
}
public static Collector<CharSequence, ?, String> joining(CharSequence delimiter,
                                                         CharSequence prefix,
                                                         CharSequence suffix) {
    return new CollectorImpl<>(
            () -> new StringJoiner(delimiter, prefix, suffix),
            StringJoiner::add, StringJoiner::merge,
            StringJoiner::toString, CH_NOID);
}
      可以看到，joining的直接拼接是通过StringBuilder来实现的；其它两种拼接都是通过StringJoiner实现的。字符串拼接相关知识可以参考这篇文章：Java字符串拼接。如下展示了三种字符串拼接方式的使用：
    // 使用Collectors.joining()拼接字符串
List<String> list = Arrays.asList("I'm","in","love","with","cocoas");
// 直接拼接
String appendStr1 = list.stream().collect(Collectors.joining());
System.out.println(appendStr1);
// 带分隔符拼接
String appendStr2 = list.stream().collect(Collectors.joining(" "));
System.out.println(appendStr2);
// 带分隔符和前后缀拼接
String appendStr3 = list.stream().collect(Collectors.joining(" ","{","}"));
System.out.println(appendStr3);
// 输出
I'minlovewithcocoas
I'm in love with cocoas
{I'm in love with cocoas}
