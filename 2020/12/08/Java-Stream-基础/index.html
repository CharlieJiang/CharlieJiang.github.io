Java Stream 基础

发表于 2020-12-08 分类于 Java基础 阅读次数：

什么是函数式编程

  简单说，函数式编程是一种编程规范，主要思想是把运算过程尽量写成一系列嵌套的函数调用。举例来说，现在有这么一个数学表达式： 

(1 + 2) * 3 -4

  传统的过程式编程可能这样写： 

int a = 1 + 2;
int b = a * 3;
int c = b -4;

  函数式编程要求使用函数，我们可以把运算过程定义成不同的函数，然后写成下面这样： 

int result = subtract(multiply(add(1,2),3), 4);

什么是Stream

  Stream是指支持顺序操作和并行聚合操作的元素序列（A sequence of elements supporting sequential and parallel aggregate operations）。其中的元素序列来自于数据源。
  Stream并不是某种数据结构，它只是数据源的一种视图。这里的数据源可以是数组、集合、I/O Channel、产生器generator等。
  Stream是Java函数式编程的主角。

如何创建Stream

  因为Stream只是数据源的一种视图，如果我们想要得到一个Stream，通常不会手动去创建，而是调用对应的工具方法。例如：

  1. 调用Collections.stream()或Collections.parallelStream()方法
    Collections.stream()示例：使用List数据源创建Stream
    // 使用List创建Stream
List<String> list = Arrays.asList("one","two","three");
list.stream().forEach(s -> System.out.println(s));
      Collections.parallelStream()：使用Set数据源创建并行Stream
    // 使用Set创建Stream
Set<Integer> set = new HashSet<>();
set.add(1);set.add(2);set.add(3);
int max = set.parallelStream().max((i,j) -> i > j ? 1 : -1).get();
System.out.println(max);
  2. 调用Arrays.stream(T[] array)方法
    使用数组创建Stream示例：
    // 使用数组创建Stream
String array[] = {"one","two","three"};
Arrays.stream(array)
        .filter(s -> s.startsWith("t"))
        .forEach(s -> System.out.println(s));
  3. 调用Stream.of(T t)/Stream.of(T… values)方法
    使用Stream.of(T t)创建只含单个元素的Stream示例：
    Stream<String> singleStream = Stream.of("cocoas");
singleStream.forEach(s -> System.out.println(s));
      使用Stream.of(T… t)创建Stream示例： 
    // 方式一：多个同类型数据
Stream<String> multiStream = Stream.of("one","two","three");
multiStream.forEach(s -> System.out.println(s));
// 方式二：数组
String array[] = {"one","two","three"};
Stream<String> multiStream2 = Stream.of(array);
multiStream2.forEach(s -> System.out.println(s));

Stream与Collections的区别

  虽然Stream一般都是通过集合的Collection.stream()方法生成，但Stream和Collection还是有以下不同：

  • 无存储：Stream不是一种数据类型，只是数据源的一种视图，并不会存储数据。
  • 函数式编程：对Stream的任何操作并不会改变其背后的数据源，比如对Stream执行过滤操作并不会删除被过滤的元素，而是生成不包含被过滤元素的新Stream。
  • 惰式执行：Stream上的操作并不会立即执行，只有当用户调用结束操作时才会执行。
  • 可消费性：Stream只能被”消费“一次，一旦遍历过后就会失效，类似容器的迭代器，想要再次遍历必须重新生成。

常见Stream接口

接口名称 接口说明
BaseStream<T,S extends BaseStream<T,S>> 流的基本接口，定义了流相关的一些基本操作。BaseStream也是其他Stream接口的父接口。
IntStream 继承自BaseStream接口，专门用于处理整形数据的Stream接口。
LongStream 继承自BaseStream接口，专门用于处理长整形数据的Stream接口。
DoubleStream 继承自BaseStream接口，专门用于处理浮点型数据的Stream接口。
Stream 继承自BaseStream接口，用于处理除以上数据类型外的其他类型数据的Stream接口。如数组、JavaBean等。

Stream<T>与JavaBean使用示例：

public class StreamClass {

    public static void main(String[] args) {
        Stream<Student> studentStream = Stream.of(new Student("cocoas"),new Student("小芳"));
        studentStream.forEach(stu -> System.out.println(stu.name));
    }

    static class Student{
        public Student(String name){
            this.name = name;
        }
        String name = "";
    }
}

// 输出
cocoas
小芳

Stream的两种操作：中间操作和结束操作

  中间操作
    中间操作都是惰式执行，调用中间操作会生成一个标记该操作的新Stream。即方法返回值为Stream类型的基本都是中间操作。
  结束操作
    结束操作才会触发真正的计算，计算发生时会把所有中间操作以pipeline的方式执行，这样可以减少迭代次数。计算完成后Stream就会失效。
  Stream中常用的中间操作和结果操作方法如下：

操作类型 操作方法
中间操作 concat() distinct() filter() flatMap() limit() map() peek() skip() sorted() parallel() sequential() unordered()
结束操作 allMatch() anyMatch() collect() count() findAny() findFirst() forEach() forEachOrdered() max() min() noneMatch() reduce() toArray()

Stream常用方法

  Stream方法使用非常依赖于函数式接口。Stream的众多方法中使用了JDK预设的一些函数式接口，包括Function<T,R>、Predicate、Supplier等。由于lambda表达式可以表示函数式接口的实现，而且实现比较简洁，所以通常都是Stream方法与lambda表达式一起使用。

  • distinct()
    方法定义： 
    Stream<T> distinct()
    方法作用：根据Object.equal()方法去除Stream中的重复元素。
    使用示例：
    List<String> list = Arrays.asList("one","two","three","one","two");
list.stream().distinct().forEach(s -> System.out.println(s));
// 输出
one
two
three
  • filter()
    方法定义：
    Stream<T> filter(Predicate<? super T> predicate)
    方法作用：过滤掉Stream中不满足predicate函数指定的条件的元素。
    使用示例：
    List<String> list = Arrays.asList("one","two","three");
list.stream()
        .filter(s -> s.length() > 3)
        .forEach(s -> System.out.println(s));
// 输出
three
  • flatMap()
    方法定义：
    <R> Stream<R> flatMap(Function<? super T, ? extends Stream<? extends R>> mapper)
    方法作用：对Stream中每个元素执行mapper函数指定的操作，每个元素执行完会产生一个新的Stream叫做Mapped Stream，然后以所有Mapped Stream中的元素组成一个新的Stream返回。简单来说，就是将原Stream中的所有元素都“摊平”后组成新的Stream，转换前后元素的个数和类型都可能会改变。
    类似方法：flatMapToDouble()、flatMapToInt()、flatMapToLong()
    使用示例：
    // 将List<String>类型的流转换为String类型的流
Stream<List<String>> listStream = Stream.of(Arrays.asList("I'm","in","love"),Arrays.asList("with","cocoas"));
Stream<String> strStream = listStream.flatMap(strList -> strList.stream());
strStream.forEach(s -> System.out.print(s + " "));
// 输出
I'm in love with cocoas
  • forEach()
    方法定义：
    void forEach(Consumer<? super T> action)
    方法作用：对Stream中每个元素执行action函数指定的操作，执行完并不会返回任何结果，常用作元素遍历。
    相似方法：forEachOrdered()
    使用示例：
    List<String> list = Arrays.asList("one","two","three");
list.stream().forEach(s -> System.out.println(s));
// 输出
one
two
three
three
  • map()
    方法定义：
    <R> Stream<R> map(Function<? super T, ? extends R> mapper)
    方法作用：对Stream中所有元素执行mapper函数所指定的操作，返回以mapper操作的结果组成的新Stream。map()相当于对元素进行转换处理，转换后元素个数不变，但元素类型可能会改变。
    相似方法：mapToDouble()、mapToInt()、mapToLong()
    使用示例：
    List<String> list = Arrays.asList("one","two","three");
// 元素类型不变
list.stream()
        .map(s -> s.toUpperCase())
        .forEach(i -> System.out.print(i + " "));
// 输出 ONE TWO THREE
// 元素类型改变
list.stream()
        .map(s -> s.length())
        .forEach(i -> System.out.print(i + " "));
// 输出 3 3 5
