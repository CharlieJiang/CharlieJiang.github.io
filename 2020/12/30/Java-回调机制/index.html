Java 回调机制

发表于 2020-12-30 分类于 Java基础 阅读次数：

模块间调用

  软件系统中，各模块间一般会存在相互调用的关系。这些调用，在调用方式上可分为三种：同步调用、异步调用和回调。

  • 同步调用

    同步调用是最基本、最简单的一种调用方式。

  同步调用过程：类A的方法a()调用类B的方法b()，方法a()一直等待方法b()执行，直到方法b()执行完毕才继续执行剩余代码。
  同步调用示例： 

public class A {
    public void a(){
        B bClass = new B();
        bClass.b();
        System.out.println("A类方法a");
    }
    public static void main(String[] args) {
        new A().a();
    }
}

public class B {
    public void b(){
        System.out.println("B类方法b");
    }
}

//输出
B类方法b
A类方法a

  同步调用存在的问题：同步调用过程中，方法a()需要等待方法b()执行完毕才能继续执行，这适用于方法b()执行耗时较短的场景。一旦方法b()执行时间过长，方法a()就会一直等待而无法继续执行，从而导致整个流程阻塞。

  • 异步调用

    异步调用是为了解决同步调用可能导致流程阻塞的问题而产生的一种调用方式。异步调用是通过创建线程进行调用的方式实现的。

  异步调用过程：类A中方法a()先创建一个线程，在线程中调用类B的方法b()，然后继续执行剩余代码。
  异步调用示例： 

public class A {
    public void a(){
        Thread th = new Thread(new Runnable() {
            @Override
            public void run() {
                B bClass = new B();
                bClass.b();
            }
        });
        th.start();
        System.out.println("A类方法a");
    }
    public static void main(String[] args) {
        new A().a();
    }
}

public class B {
    public void b(){
        try {
            Thread.sleep(1000);
        }catch (InterruptedException e){
            e.printStackTrace();
        }
        System.out.println("B类方法b");
    }
}

//输出
A类方法a
B类方法b

  由于方法a()并未等待方法b()执行完毕就继续执行剩余代码，当方法a()需要使用方法b()返回的结果时，就需要通过某种方式去监听方法b()的返回结果。比如使用Future+Callable实现返回结果监听。
  异步调用返回结果监听示例： 

public class A {

    public void a(){    //可以获取返回结果的异步调用
        ExecutorService executor = Executors.newCachedThreadPool();
        Task task = new Task();
        Future<String> result = executor.submit(task);
        try {
            Thread.sleep(1000); //模拟主线程正在执行其它任务
            System.out.println("调用类B的方法b返回的结果：" + result.get());
        } catch (InterruptedException | ExecutionException e) {
            e.printStackTrace();
        }
    }

    class Task implements Callable<String>{ //执行调用类B的方法b的任务
        @Override
        public String call() throws Exception {
            B bClass = new B();
            String result = bClass.b();
            return result;
        }
    }

    public static void main(String[] args) {
        new A().a();
    }
}

public class B {

    public String b() {
        try {
            Thread.sleep(2 * 1000); //模拟方法b执行时间
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        return "cocoas";
    }
}
  • 回调

    回调的思想是将原本方法a()被动等待方法b()执行结果的单向调用方式改为方法b()执行完后主动通知方法a()的双向调用方式。

  回调过程：类A的方法a()调用类B的方法b()，方法b()执行完毕后主动调用类A的方法c()，去通知类A方法b()执行的结果。其中，方法c()称为回调方法。
  回调示例： 

public class A {

    public void a(){
        System.out.println("A类方法a");
        B bClass = new B();
        bClass.b(this);
    }

    public void c(String result){// 类A的回调方法
        System.out.println("B类方法b返回结果：" + result);
    }

    public static void main(String[] args) {
        new A().a();
    }

}
public class B {

    public void b(A a) {
        System.out.println("B类方法b");
        try {
            Thread.sleep(2 * 1000); //模拟执行时间
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        a.c("I'm back");// 调用A类的方法a返回执行结果
    }

}
// 输出
A类方法a
B类方法b
B类方法b返回结果：I'm back

同步回调和异步回调

  与调用分同步和异步一样，回调方式也可分为同步回调和异步回调。两者的区别，主要体现在是否需要等待。
  同步调用中，因调用方的执行依赖于被调用方返回的结果，调用方需要等待被调用方返回结果后才能继续执行后续代码，等待过程中，调用方处于阻塞状态。
  异步回调中，因调用方的执行不依赖于被调用方的返回结果，调用方调用完后无需等待回调结果，即可继续执行后续代码。异步回调只需要在同步回调的基础上，在调用处加上线程即可。
  同步回调示例： 

public class A implements Callback {

    public void a(){
        B bClass = new B();
        bClass.b(this);
    }
    
    @Override
    public void callback(String result){    //类A的回调方法
        System.out.println("B类方法b返回结果：" + result);
    }

    public static void main(String[] args) {
        new A().a();
    }

}

public class B {

    public void b(Callback callback) {
        callback.callback("cocoas");
    }
}

public interface Callback {

    void callback(String result);

}

  异步回调示例： 

public class A implements Callback {

   public void a(){    //异步回调
        Thread th = new Thread(new Runnable() {
            @Override
            public void run() {
                B bClass = new B();
                bClass.b(A.this);
            }
        });
        th.start();
    }
    
    @Override
    public void callback(String result){    //类A的回调方法
        System.out.println("B类方法b返回结果：" + result);
    }

    public static void main(String[] args) {
        new A().a();
    }

}

public class B {

    public void b(Callback callback) {
        try {
            Thread.sleep(2 * 1000); //模拟执行时间
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        callback.callback("cocoas");
    }
}

public interface Callback {

    void callback(String result);

}
