RxJava原理解析

发表于 2021-05-31 分类于 Android ， RxJava ， 第三方框架原理解析 阅读次数：

RxJava是什么

RxJava是Java虚拟机的响应式扩展，是一个在Java虚拟机上使用可观测的序列来组成异步的、基于事件的程序的库。简而言之，就是一个用于处理异步事件的框架。

响应式编程：一种使用异步数据流编程的思想，RxJava是基于观察者模式的响应式编程。
链式调用：链式调用，说白了就是函数调用的结果能够执行下一次函数调用，即多个函数调用可以链式地连接在一起执行调用，而不是需要单独进行多次调用。

为什么使用RxJava，而不是AsyncTask或者Handler？

RxJava的实现更加简洁。第一，RxJava是基于事件流的方式实现链式调用，使得处理异步任务的逻辑简洁清晰。第二，RxJava基于观察者模式的事件回调函数中，除了提供一般的事件回调函数next()外，还额外提供了特殊的事件处理函数onCompleted()和onError()，以帮助处理事件的回调。

RxJava基本概念

  • Observable：被观察者，负责产生事件
  • Observer：观察者，接收事件并消费事件
  • subscribe()：订阅操作，连接被观察者和观察者
    Observable和Observer通过subscribe()实现订阅关系，当Observable产生事件时，Observer就可以接收到并消费此事件。
  • onNext()：观察者产生事件

RxJava如何实现异步

RxJava的异步是通过一种扩展的观察者模式来实现的。
RxJava中，观察者Observer通过subscribe()方法订阅到Observable上，Observable便可以在需要的时候将订阅的事件发送到Observer。不过与传统的观察者模式不同的是，RxJava的事件回调方法除了普通事件onNext()方法（相当于onClick()/onEvent()）外，还定义了两个特殊的事件：onCompleted()和onError()。

  • onCompleted()：事件队列完结
    RxJava不仅把每个事件单独处理，还会把它们看做一个队列。RxJava规定，当不会再有新的onNext()发出时，需要触发onCompleted()方法作为标志。
  • onError()：事件队列异常
    当事件处理过程中出现异常时，onError()会被触发，同时事件队列自动终止，不允许再有事件发出。

自由的线程切换

默认单线程

默认情况下，RxJava是单线程的，只在当前线程中执行。此时，Observable用于发射数据流，Observer用于接收和响应数据流，各种操作符Operators用于加工数据流，它们都在同一个线程中运行。

使用调度器Scheduler切换线程

RxJava在链式调用的基础上，通过设置不同的调度器Scheduler，可以灵活的在不同线程间切换并执行对应的任务。比如先切换到子线程中执行耗时的任务，然后再切回主线程更新页面。
RxJava提供便捷的线程切换功能，通过执行subscribeOn和observeOn操作即可轻松地实现线程切换。基于便捷的线程切换功能，RxJava也常常会被用于代替Android原生的AsyncTask、Handler等来执行异步任务。

调度器Scheduler

Scheduler相当于线程调度器，是所有RxJava调度器的抽象父类，子类需要实现其createWorker()方法，返回一个Worker实例，用于接受并执行Task。同时也可以重写其scheduleDirect()方法决定如何将Task分配给不同的Worker。RxJava通过Scheduler让操作跑在指定的线程，从而实现多线程调度。

RxJava内置了多个Scheduler的实现，它们能满足大多数使用场景。例如：

  • Scheduler.single：使用定长为1的线程池，并重复利用这个线程。
  • Scheduler.newThread：每次都启用新线程，并在新线程中执行操作。
  • Scheduler.computation：使用固定的线程池，线程池大小为CPU的核数 ，适用于CPU密集型计算。
  • Scheduler.io：适用I/O操作（读写文件、数据库、网络数据等），行为模式与newThread类似，区别在于io的内部实现是使用一个无数量上线的线程池，并可以重用空闲的线程，因此多数情况下，io的效率会比newThread要高。
  • Scheduler.trampoline：直接在当前线程执行，如果当前线程中其它任务在执行，则会先暂停其它任务。
  • Scheduler.from：将java.util.concurrent.Executor转换成一个Scheduler实例，即可以自定义一个Executor来作为调度器。
  • AndroidScheduler.mainThread：Android专用的一个调度器，它指定的操作将会在Android的主线程中执行。

线程切换具体实现方式

  • subscribeOn()

    subscribeOn()指定subscribe()所发生的线程，或者叫作事件产生的线程。
    如果调用多次，则只有第一次会起作用。

实现原理：
创建一个实现了Runnable接口的线程类SubscribeTask，在SubscribeTask的run()方法中调用Observable.subscribe()，然后调用调度器Scheduler的scheduleDirect()方法进行线程切换。相关源码如下：
ObservableSubscribeOn.SubscribeTask线程的run()中执行订阅操作：

final class SubscribeTask implements Runnable {
    private final SubscribeOnObserver<T> parent;

    SubscribeTask(SubscribeOnObserver<T> parent) {
        this.parent = parent;
    }

    @Override
    public void run() {
        source.subscribe(parent);
    }
}

ObservableSubscribeOn的subscribeActual()中将订阅处理线程SubscribeTask交由线程调度器Scheduler进行线程切换处理： 

public void subscribeActual(final Observer<? super T> observer) {
    final SubscribeOnObserver<T> parent = new SubscribeOnObserver<>(observer);

    observer.onSubscribe(parent);

    parent.setDisposable(scheduler.scheduleDirect(new SubscribeTask(parent)));
}
  • observeOn()

    指定观察者Observer所在的线程，或者叫做事件消费的线程。
    如果调用多次，则每次都会起作用。

实现原理：
先基于Scheduler.createWorker()方法创建一个新的Worker，再创建一个实现了Runnable和Observer接口的线程类ObserveonObserver，通过在其事件回调方法onNext()、onComplete()、onError()等中调用Worker.schedule()方法实现线程切换。相关源码如下：
ObservableObserveOn.subscribeActual()方法中创建Scheduler.Worker对象用于线程切换：

protected void subscribeActual(Observer<? super T> observer) {
    if (scheduler instanceof TrampolineScheduler) {
        source.subscribe(observer);
    } else {
        Scheduler.Worker w = scheduler.createWorker();

        source.subscribe(new ObserveOnObserver<>(observer, w, delayError, bufferSize));
    }
}

ObservableObserveOn.ObserveOnObserver线程类中定义了一个schedule()方法，在其中通过调用Worker.schedule()实现切换线程。此方法由ObserveOnObserver类的事件回调方法调用，即有事件回调时触发线程的切换。

static final class ObserveOnObserver<T> extends BasicIntQueueDisposable<T>
    implements Observer<T>, Runnable {
    ...
    @Override
    public void onNext(T t) {
        ...
        schedule();
    }

    @Override
    public void onError(Throwable t) {
        ...
        schedule();
    }

    @Override
    public void onComplete() {
        ...
        schedule();
    }

    void schedule() {
        if (getAndIncrement() == 0) {
            worker.schedule(this);
        }
    }
    ...
}

切换线程示例

Observable.from(new String[]{"one","two","three"})
      .subscribeOn(Schedulers.io()) // 指定事件分发所在线程
      .observeOn(AndroidSchedulers.mainThread()) // 指定事件消费所在线程
      .subscribe(str -> System.out.println(str)); // 事件消费

事件序列变换

类似Java 8 的Stream API，对事件进行流式加工。如map()、filter()等操作。

适用场景

  • 与Retrofit结合：代码逻辑清晰、简捷明了
  • RxBinding：实现Android平台的Binding API，绑定Android平台上的控件，并以Observable的形式提供事件反馈等。
  • 执行异步操作：替代AsyncTask、Handler等
  • 实现事件总线：替代otto、EventBus等

RxJava 2.0 新特性

依赖独立

RxJava 2.0的依赖是全新的io.reactivex.rxjave2:rxjava:2.x.x，1.0的则是io.reactivex:rxjava:x.y.z。

新增基础接口

  • Publisher：事件生产者接口
  • Subscriber：事件订阅者接口
  • Subscription：用于处理背压的接口
    Subscription接口提供了request(long n)和cancel()两个方法，使得Subscriber可以主动的决定是向上游请求数据，还是打断事件流并释放资源。
  • Processor：Publisher + Subscriber，且支持背压

支持背压Backpressure

背压是指在异步场景中，被观察者发送事件速度远快于观察者处理事件速度的情况。

RxJava 1.x中Observable可以使用背压策略支持背压。RxJava 2.x中，原来的Observable被分成了新的Observable和Flowable。新的Observable用于订阅Observer，不支持背压；Flowable实现了Publisher接口，用于订阅Subscriber，支持背压。

RxJava与RxAndroid

RxAndroid是一个RxJava针对Android平台的扩展，主要用于Android开发。

RxJava与Java 8 Stream

  • Stream是同步、阻塞的，RxJava是异步、非阻塞的。
  • 流只能使用一次，Observable可以多次订阅
  • 并行流无法指定使用的线程池，RxJava可以通过Scheduler设定多种线程池

相关知识

Promise

Promise是JavaScript中异步任务处理的对象。支持基于事件流的任务处理。

java.util.concurrent.Executor

  • 是什么：Executor是Java提供的多线程框架
  • 干什么：管理线程池、调度和执行异步任务、异步任务结果反馈
  • 怎么用：
  1. 使用Executors创建线程池：ExecutorService threadPool = Executors.newFixedThreadPool(4);
  2. 创建异步任务Task：Thread task = new Thread(){...run(){//to do something}}
  3. 提交任务到线程池：Future<String> taskResult = threadPool.submit(task);
  4. 线程池调度执行任务并提供执行结果反馈：String result = taskResult.get();
  5. 关闭线程池：threadPool.shutdown();

Executor线程池种类
单线程池SingleThreadExecutor：顺序执行任务，线程dead后重新创建线程进行替换。
固定数量线程池FixedThreadPool(n)：线程可复用， 线程池满后，新的线程需要在队列中等待。
缓存线程池CachedThreadPool：适用执行大量短期任务；线程数量无限制，需要即创建；线程空闲超过60秒即会被从缓存中删除。
计划型线程池Scheduledexecutor：执行定期任务
