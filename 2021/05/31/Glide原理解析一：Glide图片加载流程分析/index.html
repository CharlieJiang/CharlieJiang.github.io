Glide原理解析一：Glide图片加载流程分析

发表于 2021-05-31 更新于 2021-06-01 分类于 Glide ， Android ， 第三方框架原理解析 阅读次数：

什么是Glide

Glide是Google官方推荐的一款图片加载框架，也是目前非常流行的Android图片加载框架。
Glide支持加载多种图片资源，包括网络图片、本地图片、资源文件图片、图片二进制流等。同时，Glide还支持GIF和本地视频显示。

如何使用Glide加载图片

一般情况下，使用Glide加载图片的方式如下： 

Glide.with(context)// 初始化Glide
        .load(url)// 加载资源
        .into(view);// 填充视图

下面将针对Glide加载图片的这三个过程进行分析。

with()方法：创建请求管理器RequestManager

with()方法的目的就是创建一个RequestManager对象，实际的创建操作交由RequestManagerRetriever完成。RequestManager负责管理、启动图片加载的相关请求，会与Activity/Fragment的生命周期进行绑定。
由于Glide是不需要提前初始化，所以当在APP中第一次使用Glide时，会通过with()方法进行初始化处理。

with()方法有多个同名方法，可以接收Activity、Fragment、FragmentActivity、View等进行Glide初始化处理。 

// Glide.java
public static RequestManager with(@NonNull Context context) 
public static RequestManager with(@NonNull Activity activity)
public static RequestManager with(@NonNull Fragment fragment) 
public static RequestManager with(@NonNull FragmentActivity activity) 
public static RequestManager with(@NonNull View view)

这里以with(activity)方法为例进行分析。with(activity)源码如下：

// Glide.java
public static RequestManager with(@NonNull Activity activity) {
  return getRetriever(activity).get(activity);
}

with(Activity)方法返回的是一个RequestManager（Request在Glide中表示的是加载图片等资源到目标视图上的这一请求，与一般的网络请求有所区别），RequestManager负责管理并启动Request。继续看getRetriever()方法：

// Glide.java
private static RequestManagerRetriever getRetriever(@Nullable Context context) {
    // Context null check
    ...
    return Glide.get(context).getRequestManagerRetriever();
}

getRetriever()方法中，先是通过Glide.get()方法去获取Glide的一个单例，然后再调用Glide.getRequestManagerRetriever()方法获取一个RequestManagerRetriever。RequestManagerRetriever是RequestManager的管理类，负责创建RequestManager，或从Activity/Fragment中检索出已存在的RequestManager。继续看Glide.get()方法：

// Glide.java
public static Glide get(@NonNull Context context) {
    if (glide == null) {
      // 根据@GlideModule注解自动生成的一个AppGlideModule：用于下面初始化Glide
      GeneratedAppGlideModule annotationGeneratedModule =
          getAnnotationGeneratedGlideModules(context.getApplicationContext());
      synchronized (Glide.class) {
        if (glide == null) { // 初始化Glide
          checkAndInitializeGlide(context, annotationGeneratedModule);
        }
      }
    }
    return glide;
}

get()方法中会判断是否已存在Glide单例对象：存在则直接返回；不存在则构建后返回。get()方法中依次通过调用checkAndInitializeGlide()、initializeGlide()方法进行Glide初始化:

// Glide.java
 private static void checkAndInitializeGlide(
      @NonNull Context context, @Nullable GeneratedAppGlideModule generatedAppGlideModule) {
    ...
    isInitializing = true;
    initializeGlide(context, generatedAppGlideModule);
    isInitializing = false;
}
  
private static void initializeGlide(
      @NonNull Context context, @Nullable GeneratedAppGlideModule generatedAppGlideModule) {
    initializeGlide(context, new GlideBuilder(), generatedAppGlideModule);
  }  

private static void initializeGlide(
      @NonNull Context context,
      @NonNull GlideBuilder builder,
      @Nullable GeneratedAppGlideModule annotationGeneratedModule) {
    Context applicationContext = context.getApplicationContext();
    ...
    // 使用GlideBuilder构建Glide
    Glide glide = builder.build(applicationContext);
    ... 
    // 构建的Glide赋值给单例对象
    Glide.glide = glide;
}

从initializeGlide()方法中可以看到，Glide是通过GlideBuilder.build()构建的。build()方法源码如下：

// GlideBuilder.java
Glide build(@NonNull Context context) {
    if (sourceExecutor == null) {// 网络资源请求任务线程池
      sourceExecutor = GlideExecutor.newSourceExecutor();
    }

    if (diskCacheExecutor == null) {// 磁盘缓存任务线程池
      diskCacheExecutor = GlideExecutor.newDiskCacheExecutor();
    }

    if (animationExecutor == null) {// 解析GIF图片任务线程池
      animationExecutor = GlideExecutor.newAnimationExecutor();
    }

    if (memorySizeCalculator == null) {// 内存缓存大小计算器
      memorySizeCalculator = new MemorySizeCalculator.Builder(context).build();
    }

    if (connectivityMonitorFactory == null) {// 网络活动监视器工厂类
      connectivityMonitorFactory = new DefaultConnectivityMonitorFactory();
    }

    if (bitmapPool == null) {// Bitmap缓存池：用于复用Bitmap
      int size = memorySizeCalculator.getBitmapPoolSize();
      if (size > 0) {
        bitmapPool = new LruBitmapPool(size);
      } else {
        bitmapPool = new BitmapPoolAdapter();
      }
    }

    if (arrayPool == null) {
      arrayPool = new LruArrayPool(memorySizeCalculator.getArrayPoolSizeInBytes());
    }

    if (memoryCache == null) {// 基于LRU的资源内存缓存
      memoryCache = new LruResourceCache(memorySizeCalculator.getMemoryCacheSize());
    }

    if (diskCacheFactory == null) {// 磁盘缓存工厂类
      diskCacheFactory = new InternalCacheDiskCacheFactory(context);
    }

    if (engine == null) {// Glide的资源加载引擎
      engine =
          new Engine(
              memoryCache,
              diskCacheFactory,
              diskCacheExecutor,
              sourceExecutor,
              GlideExecutor.newUnlimitedSourceExecutor(),
              animationExecutor,
              isActiveResourceRetentionAllowed);
    }

    if (defaultRequestListeners == null) {
      defaultRequestListeners = Collections.emptyList();
    } else {
      defaultRequestListeners = Collections.unmodifiableList(defaultRequestListeners);
    }

    GlideExperiments experiments = glideExperimentsBuilder.build();
    RequestManagerRetriever requestManagerRetriever =// RequestManager的管理器
        new RequestManagerRetriever(requestManagerFactory, experiments);

    return new Glide(// 创建Glide对象
        context,
        engine,
        memoryCache,
        bitmapPool,
        arrayPool,
        requestManagerRetriever,
        connectivityMonitorFactory,
        logLevel,
        defaultRequestOptionsFactory,
        defaultTransitionOptions,
        defaultRequestListeners,
        experiments);
  }

GlideBuilder.build()方法中通过众多的初始化配置后，直接new了一个Glide对象返回。到这里，我们已经知道了Glide单例的大概创建过程。虽然最后的build()方法中进行了不少的配置，不过真正的繁杂配置其实在Glide的构造函数中，其中通过Registry注册了大量的组件，用于编解码及加载资源，这里暂不分析。
Glide()的初始化过程分析完，我们返回看下with()方法中获取RequestManager的方法RequestManagerRetriever.get()。get()方法与with()方法对应有多个同名方法：

// RequestManagerRetriever.java
public RequestManager get(@NonNull Context context) 
public RequestManager get(@NonNull Activity activity)
public RequestManager get(@NonNull Fragment fragment)
public RequestManager get(@NonNull FragmentActivity activity)
public RequestManager get(@NonNull View view)

以get(activity)为例：

// RequestManagerRetriever.java
public RequestManager get(@NonNull Activity activity) {
  if (Util.isOnBackgroundThread()) {
      // 查找Application类型的RequestManager
    return get(activity.getApplicationContext());
  } else if (activity instanceof FragmentActivity) {
      // 查找FragmentActivity类型的RequestManager
    return get((FragmentActivity) activity);
  } else {
    assertNotDestroyed(activity);
    frameWaiter.registerSelf(activity);
    // 基于FragmentManager查找RequestManager
    android.app.FragmentManager fm = activity.getFragmentManager();
    return fragmentGet(activity, fm, /*parentHint=*/ null, isActivityVisible(activity));
  }
}

private RequestManager fragmentGet(
    @NonNull Context context,
    @NonNull android.app.FragmentManager fm,
    @Nullable android.app.Fragment parentHint,
    boolean isParentVisible) {
        //构建RequestManagerFragment查找RequestManager
  RequestManagerFragment current = getRequestManagerFragment(fm, parentHint);
  RequestManager requestManager = current.getRequestManager();
  // 从FragmentManager中未找到RequestManager，要重新创建
  if (requestManager == null) {
    // TODO(b/27524013): Factor out this Glide.get() call.
    Glide glide = Glide.get(context);
    // 通过RequestManagerFactory构建新的RequestManager
    requestManager =
        factory.build(
            glide, current.getGlideLifecycle(), current.getRequestManagerTreeNode(), context);
    if (isParentVisible) {
      requestManager.onStart();
    }
    // RequestManager绑定到RequestManagerFragment上，方便下次查找
    current.setRequestManager(requestManager);
  }
  return requestManager;
}

get()方法的作用就是：依次尝试查找Application类型RequestManager、FragmentActivity类型RequestManager、最后尝试从FragmentManager中查找RequestManager，如果没有找到则会使用RequestManagerFactory.build()重新创建一个RequestManager。build()方法内的current.getGlideLifecycle()返回的是一个ActivityFragmentLifecycle，通过ActivityFragmentLifecycle，RequestManager就能获知Activity/Fragment的生命周期事件。

// RequestManagerFragment.java
ActivityFragmentLifecycle getGlideLifecycle() {
    return lifecycle;
}

load()：创建请求构造器RequestBuilder加载资源

load()是RequestManager中的方法，load()返回的是一个RequestBuilder，其负责创建Request。RequestManager中的大多数方法都是负责链式配置RequestBuilder的各种属性。load(String)源码如下：

// RequestManager.java
public RequestBuilder<Drawable> load(@Nullable String string) {
  return asDrawable().load(string);
}

public RequestBuilder<Drawable> asDrawable() {
  return as(Drawable.class);
}

public <ResourceType> RequestBuilder<ResourceType> as(
    @NonNull Class<ResourceType> resourceClass) {
  return new RequestBuilder<>(glide, this, resourceClass, context);
}

as()方法的作用，就是构建指定资源类型的RequestBuilder。比如Drawable、Bitmap、GifDrawable。asDrawable()方法通过as(Drawable.class)构建了Drawable类型的RequestBuilder：RequestBuilder<Drawable>，然后又链式调用了RequestBuilder的load()方法：

// RequestManager.java
public RequestBuilder<TranscodeType> load(@Nullable String string) {
  return loadGeneric(string);
}

private RequestBuilder<TranscodeType> loadGeneric(@Nullable Object model) {
  if (isAutoCloneEnabled()) {
    return clone().loadGeneric(model);
  }
  this.model = model;
  isModelSet = true;
  return selfOrThrowIfLocked();
}

RequestBuilder.load()方法比较简单，就是设置下加载数据的model参数，model参数类型可以是String、Uri、Bitmap等。

into()：资源加载到指定视图

先看下RequestBuilder.into(ImageView)方法的源码：

// RequestBuilder.java
public ViewTarget<ImageView, TranscodeType> into(@NonNull ImageView view) {
  Util.assertMainThread();
  Preconditions.checkNotNull(view);
    // 获取Request的基础配置信息
  BaseRequestOptions<?> requestOptions = this;
  // 根据scale type设置裁剪策略
  if (!requestOptions.isTransformationSet()
      && requestOptions.isTransformationAllowed()
      && view.getScaleType() != null) {
    // Clone in this method so that if we use this RequestBuilder to load into a View and then
    // into a different target, we don't retain the transformation applied based on the previous
    // View's scale type.
    switch (view.getScaleType()) {
      case CENTER_CROP:
        requestOptions = requestOptions.clone().optionalCenterCrop();
        break;
      case CENTER_INSIDE:
        requestOptions = requestOptions.clone().optionalCenterInside();
        break;
      case FIT_CENTER:
      case FIT_START:
      case FIT_END:
        requestOptions = requestOptions.clone().optionalFitCenter();
        break;
      case FIT_XY:
        requestOptions = requestOptions.clone().optionalCenterInside();
        break;
      case CENTER:
      case MATRIX:
      default:
        // Do nothing.
    }
  }

  return into(
      glideContext.buildImageViewTarget(view, transcodeClass),
      /*targetListener=*/ null,
      requestOptions,
      Executors.mainThreadExecutor());
}

into(ImageView)方法完成三个功能：

  • 获取Request配置信息BaseRequestOptions
  • 设置图片裁剪策略
  • 调用多参数into()方法继续执行后续操作。

BaseRequestOptions中保存着RequestManager的所有配置信息，包括缓存策略、裁剪大小、动画效果、优先级等，用于创建Request时配置其属性。
glideContext.buildImageViewTarget()方法构建了一个ViewTarget，ViewTarget的继承关系如下：ViewTarget——BaseTarget——Target。Glide中的Target负责资源的加载，并通过继承LifeCycleListener实现对Fragment/Activity生命周期事件的监听。BaseTarget通过setRequest()绑定请求实现了加载过程的生命周期事件监听及处理，如onLoadStarted、onResourceReady、onLoadFailed等。
buildImageViewTarget()的源码如下：

// GlideContext.java
public <X> ViewTarget<ImageView, X> buildImageViewTarget(

  return imageViewTargetFactory.buildTarget(imageView, transcodeClass);

  }

ViewTarget由ImageViewTargetFactory创建，ImageViewTargetFactory会根据资源类型创建BitmapImageViewTarget或是DrawableImageViewTarget。
继续看最后调用的多参数into()方法，其源码如下：

// GlideContext.java
private <Y extends Target<TranscodeType>> Y into(
    @NonNull Y target,
    @Nullable RequestListener<TranscodeType> targetListener,
    BaseRequestOptions<?> options,
    Executor callbackExecutor) {
  Preconditions.checkNotNull(target);
  if (!isModelSet) {
    throw new IllegalArgumentException("You must call #load() before calling #into()");
  }
    // 创建一个新的请求
  Request request = buildRequest(target, targetListener, options, callbackExecutor);
    // 获取target上一次的请求
  Request previous = target.getRequest();
  // 判断使用新的请求，还是执行之前的请求
  if (request.isEquivalentTo(previous)// 两个请求相等
      && !isSkipMemoryCacheWithCompletePreviousRequest(options, previous)) {// 当前请求不缓存，之前请求已完成
    if (!Preconditions.checkNotNull(previous).isRunning()) {
      previous.begin();
    }
    return target;
  }
  
  requestManager.clear(target);// 先取消跟踪target
  target.setRequest(request);// target绑定新request
  requestManager.track(target, request);// 跟踪target并启动其新的request

  return target;
}

buildRequest()方法创建了一个新的Request，Request的类型有三种：

  • ErrorRequestCoordinator：错误图片+目标图片的联合请求
  • ThumbnailRequestCoordinator：缩略图+目标图片的联合请求
  • SingleRequest：加载目标图片的单一请求
    target.getRequest()方法用于获取target之前绑定的request，其源码如下：
    //ViewTarget.java
public Request getRequest() {
    Object tag = getTag();
    Request request = null;
    if (tag != null) {
        if (tag instanceof Request) {
            request = (Request) tag;
        } else {
            throw new IllegalArgumentException("You must not call setTag() on a view Glide is targeting");
        }
    }
    return request;
}

private Object getTag() {
  return view.getTag(tagId);
}
    可以看到，Glide实际上是把Request放进了view的tag属性中，以方便存取。
    RequestManager.track()跟踪Target并执行其请求，track()源码如下： 
    // RequestManager.java
synchronized void track(@NonNull Target<?> target, @NonNull Request request) {
  targetTracker.track(target);// 将Target加入跟踪列表
  requestTracker.runRequest(request); // 将request加入请求跟踪队列并启动请求
}
    TargetTracker内部持有所有活动Target的集合，并负责在收到生命周期事件时通知所有的Target。相当于Target向TargetTracker注册成为观察者。RequestTracker内部持有Request集合，负责跟踪并管理Request，如取消、启动Request等。runRequest()源码如下：
    // RequestTracker.java
public void runRequest(@NonNull Request request) {
  requests.add(request);
  if (!isPaused) {
    request.begin();
  } else {
    request.clear();
    if (Log.isLoggable(TAG, Log.VERBOSE)) {
      Log.v(TAG, "Paused, delaying request");
    }
    pendingRequests.add(request);
  }
}
    以SingleRequest为例，看下Request.begin()源码：
    // SingleRequest.java
public void begin() {
  synchronized (requestLock) {
    assertNotCallingCallbacks();
    stateVerifier.throwIfRecycled();
    startTime = LogTime.getLogTime();
    if (model == null) {// 要加载的资源为空，调用加载失败回调
      if (Util.isValidDimensions(overrideWidth, overrideHeight)) {
        width = overrideWidth;
        height = overrideHeight;
      }

      int logLevel = getFallbackDrawable() == null ? Log.WARN : Log.DEBUG;
      onLoadFailed(new GlideException("Received null model"), logLevel);
      return;
    }

    if (status == Status.RUNNING) {// 重复请求，直接抛出异常
      throw new IllegalArgumentException("Cannot restart a running request");
    }

    if (status == Status.COMPLETE) {// 重复请求已完成，可以回调加载资源
      onResourceReady(
          resource, DataSource.MEMORY_CACHE, /* isLoadedFromAlternateCacheKey= */ false);
      return;
    }

    // 重复请求未运行或未完成，可以像新请求一样重新执行。
    status = Status.WAITING_FOR_SIZE;
    if (Util.isValidDimensions(overrideWidth, overrideHeight)) {
      onSizeReady(overrideWidth, overrideHeight);// 拿到size，可以开始异步请求
    } else {
      target.getSize(this);// 从target中获取size（获取到size后还是会调用onSizeReady()）
    }

    if ((status == Status.RUNNING || status == Status.WAITING_FOR_SIZE)
        && canNotifyStatusChanged()) {
      target.onLoadStarted(getPlaceholderDrawable());
    }
    if (IS_VERBOSE_LOGGABLE) {
      logV("finished run method in " + LogTime.getElapsedMillis(startTime));
    }
  }
}
    再看下其中的onSizeReady()方法：
    public void onSizeReady(int width, int height) {
  stateVerifier.throwIfRecycled();// 判断当前的同步调用方法是否已被释放
  synchronized (requestLock) {
    if (IS_VERBOSE_LOGGABLE) {
      logV("Got onSizeReady in " + LogTime.getElapsedMillis(startTime));
    }
    if (status != Status.WAITING_FOR_SIZE) {
      return;
    }
    status = Status.RUNNING;

    float sizeMultiplier = requestOptions.getSizeMultiplier();// 缩略图的图片尺寸系数（0~1）
    // 根据图片尺寸系数，计算图片的长宽    
    this.width = maybeApplySizeMultiplier(width, sizeMultiplier);
    this.height = maybeApplySizeMultiplier(height, sizeMultiplier);

    if (IS_VERBOSE_LOGGABLE) {
      logV("finished setup for calling load in " + LogTime.getElapsedMillis(startTime));
    }
    loadStatus = // 基于Engine加载资源
        engine.load(
            glideContext,
            model,
            requestOptions.getSignature(),
            this.width,
            this.height,
            requestOptions.getResourceClass(),
            transcodeClass,
            priority,
            requestOptions.getDiskCacheStrategy(),
            requestOptions.getTransformations(),
            requestOptions.isTransformationRequired(),
            requestOptions.isScaleOnlyOrNoTransform(),
            requestOptions.getOptions(),
            requestOptions.isMemoryCacheable(),
            requestOptions.getUseUnlimitedSourceGeneratorsPool(),
            requestOptions.getUseAnimationPool(),
            requestOptions.getOnlyRetrieveFromCache(),
            this,
            callbackExecutor);

    if (status != Status.RUNNING) {
      loadStatus = null;
    }
    if (IS_VERBOSE_LOGGABLE) {
      logV("finished onSizeReady in " + LogTime.getElapsedMillis(startTime));
    }
  }
}
